# 部署说明

## 环境要求
- **后端**：Node.js v24.12.0
- **数据库**：MySQL 8+
- **前端**：
  - 微信开发者工具（用于小程序调试）
  - Nginx（用于管理后台前端部署）
- **容器**：Docker（推荐，用于后端服务部署）
- **网络**：HTTPS 支持

---

## 环境变量

### 数据库
- DB_HOST
- DB_PORT（默认 3306）
- DB_USER
- DB_PASS
- DB_NAME

### 微信
- WX_APPID
- WX_SECRET

### 安全
- JWT_SECRET
- BCRYPT_ROUNDS（默认 10）

### 日志
- LOG_LEVEL（debug/info/warn/error，默认 info）
- LOG_PATH（默认 /logs）

### 其他
- NODE_ENV（development/test/production）
- PORT（默认 8902）
- CORS_ORIGIN（允许的跨域域名，多个用逗号分隔）

## 部署方式

### 1. 后端服务（ucar-api）

#### 使用 Docker 部署
```bash
# 进入后端目录
cd ucar-api

# 构建 Docker 镜像
docker build -t ucar-api .

# 运行容器
docker run -d \
  -p 8902:8902 \
  --env-file .env \
  --name ucar-api \
  ucar-api
```

#### 环境变量配置（.env 文件）
```
# 服务器配置
PORT=8902
NODE_ENV=production

# 验证码配置（生产环境建议启用）
CAPTCHA_ENABLED=true

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASS=password
DB_NAME=ucar_system

# 微信配置
WX_APPID=wx1234567890abcdef
WX_SECRET=your_wx_secret

# 安全配置
JWT_SECRET=your_jwt_secret
BCRYPT_ROUNDS=10

# 其他配置
CORS_ORIGIN=*
```

### 2. 管理后台前端（ucar-admin-web）

#### 构建
```bash
# 进入前端目录
cd ucar-admin-web

# 安装依赖
npm install

# 构建生产版本
npm run build
```

#### 部署到 Nginx
```nginx
server {
  listen 80;
  server_name admin.example.com;
  
  # 重定向到 HTTPS
  return 301 https://$host$request_uri;
}

server {
  listen 443 ssl;
  server_name admin.example.com;
  
  # SSL 配置
  ssl_certificate /path/to/cert.pem;
  ssl_certificate_key /path/to/key.pem;
  ssl_protocols TLSv1.2 TLSv1.3;
  
  # 静态资源
  root /var/www/ucar-admin-web/dist;
  index index.html;
  
  # 处理 SPA 路由
  location / {
    try_files $uri $uri/ /index.html;
  }
  
  # API 代理
  location /api/ {
    proxy_pass http://localhost:8902/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

### 3. 微信小程序（ucar-miniapp）

#### 准备工作
1. **注册微信小程序账号**
   - 访问 [微信公众平台](https://mp.weixin.qq.com/)
   - 注册小程序账号
   - 获取 AppID

2. **配置服务器域名**
   - 登录微信公众平台
   - 进入「开发」→「开发设置」
   - 配置服务器域名：
     - request 合法域名：`https://your-api-domain.com`
     - socket 合法域名：（如需）
     - uploadFile 合法域名：（如需）
     - downloadFile 合法域名：（如需）

3. **安装依赖**
```bash
# 进入小程序目录
cd ucar-miniapp

# 安装依赖
npm install

# 编译样式
npm run unocss
```

4. **配置 API 地址**
   修改 `app.js` 中的 `baseUrl`：
   ```javascript
globalData: {
  userInfo: null,
  token: null,
  baseUrl: 'https://your-api-domain.com',
}
```

5. **导入项目到微信开发者工具**
   - 打开微信开发者工具
   - 选择「导入项目」
   - 选择 `ucar-miniapp` 目录
   - 填写小程序 `AppID`
   - 选择「不使用云服务」
   - 点击「导入」

6. **测试与调试**
   - 在微信开发者工具中点击「编译」
   - 使用模拟器预览效果
   - 真机调试（扫描二维码）

7. **上传与发布**
   - 点击「上传」按钮
   - 填写版本号和更新说明
   - 提交审核
   - 审核通过后发布

#### 小程序部署注意事项
- **代码包大小**：控制在 2MB 以内，超过需使用分包
- **安全配置**：确保所有接口使用 HTTPS
- **隐私政策**：小程序必须包含隐私政策页面
- **审核规范**：
  - 确保功能符合微信小程序审核规范
  - 避免使用违规内容
  - 确保用户数据安全
- **版本管理**：
  - 每次更新使用递增的版本号
  - 记录版本更新内容

## 日志规范

### 日志格式
统一使用 JSON 格式，包含以下字段：
```json
{
  "timestamp": "2026-04-22T10:30:00.000Z",
  "level": "info",
  "trace_id": "uuid",
  "module": "lead",
  "message": "新增线索成功",
  "user_id": "admin001",
  "ip": "192.168.1.1",
  "duration_ms": 45,
  "extra": {}
}
```

### 日志级别
- `debug`：开发调试信息，生产环境不输出
- `info`：正常业务日志（接口请求、状态变更）
- `warn`：警告信息（频率限制触发、非关键异常）
- `error`：错误信息（数据库连接失败、接口异常）

### 日志切割
- 按日期切割：`<LOG_PATH>/app-YYYY-MM-DD.log`
- 单文件最大 100MB，保留最近 30 天
- 错误日志单独存储：`<LOG_PATH>/error-YYYY-MM-DD.log`

## 监控指标

### 系统监控（Prometheus）
- CPU 使用率、内存使用率、磁盘使用率
- Node.js 事件循环延迟、GC 频率
- MySQL 连接数、慢查询数量

### 业务监控
- 接口 QPS、P50/P95/P99 响应时间
- 错误率（按接口维度统计）
- 登录成功率、微信登录成功率
- 线索转化率 = 成交数 / 线索总数
- 结算完成率 = 已结算数 / 结算总数

### 告警规则
- 接口 P99 响应时间 > 1000ms，持续 5 分钟
- 错误率 > 5%，持续 3 分钟
- 数据库连接数 > 80% 最大连接数
- 磁盘使用率 > 85%
- 登录接口失败率 > 20%，持续 5 分钟

## 安全配置
- **HTTPS**：所有接口必须通过 HTTPS
- **CORS**：配置跨域，仅允许指定域名访问
- **数据库**：数据库账号需最小权限原则
- **密码**：使用 bcrypt 加密存储
- **JWT**：使用安全的密钥，设置合理的过期时间
- **输入验证**：所有用户输入必须经过验证
- **日志**：不记录敏感信息

## RSA密钥对配置

### 生成方法
```bash
# 在ucar-api目录下执行
openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -pubout -out public.pem
```

### 文件放置位置
- `public.pem`：放置在ucar-api根目录，用于密码加密
- `private.pem`：妥善保管，用于解密查看用户真实密码（仅管理员使用）

### 作用
- **public.pem**：系统使用公钥对用户密码进行RSA加密，存储到数据库的`password_secure`字段
- **private.pem**：管理员使用私钥解密`password_secure`字段，查看用户真实密码

### 注意事项
- 私钥文件必须妥善保管，避免泄露
- 每次部署时确保公钥文件存在于正确位置
- 不要将私钥文件提交到版本控制系统

## 数据库备份与恢复

### 备份策略
- 每日凌晨 2:00 自动全量备份
- 备份文件命名：`backup-YYYY-MM-DD-HH-mm-ss.sql`
- 存储路径：`/backup`
- 保留策略：保留最近 7 天全量备份
- 备份文件大小超过 1GB 时进行压缩存储（.gz）

### 恢复方式
```bash
# 恢复全量备份
mysql -u user -p db_name < backup-2026-04-22-02-00-00.sql

# 恢复压缩备份
gunzip < backup-2026-04-22-02-00-00.sql.gz | mysql -u user -p db_name
```

### 回滚方案
- 部署前创建数据库快照
- 应用升级失败时，回滚到上一个版本的 Docker 镜像
- 数据库变更使用迁移脚本（migration），支持回滚

## 常见问题

### 1. 微信登录失败
- **原因**：AppID 或 AppSecret 配置错误
- **解决**：检查环境变量中的 WX_APPID 和 WX_SECRET

### 2. 小程序无法访问 API
- **原因**：服务器域名未配置
- **解决**：在微信公众平台配置 request 合法域名

### 3. 数据库连接失败
- **原因**：数据库配置错误
- **解决**：检查环境变量中的数据库配置

### 4. 部署后页面空白
- **原因**：Nginx 配置错误
- **解决**：检查 Nginx 配置，确保 SPA 路由处理正确

### 5. 权限问题
- **原因**：文件权限或 Docker 容器权限
- **解决**：检查文件权限，确保容器有正确的访问权限

### 6. 本地开发登录报"验证码错误或已过期"
- **原因**：后端使用内存存储验证码（`captchaStore`），`nodemon` 热重载会清空内存
- **解决**：开发环境在 `.env` 中设置 `CAPTCHA_ENABLED=false` 禁用验证码验证
- **注意**：生产环境务必保持 `CAPTCHA_ENABLED=true`

# 部署说明

## 环境要求
- Node.js v24.12.0
- MySQL 8+
- 微信开发者工具（用于小程序调试）
- Docker（推荐，用于后端服务部署）
- Nginx（用于前端静态资源部署）

---

## 环境变量

### 数据库
- DB_HOST
- DB_PORT（默认 3306）
- DB_USER
- DB_PASS
- DB_NAME

### 微信
- WX_APPID
- WX_SECRET

### 安全
- JWT_SECRET
- BCRYPT_ROUNDS（默认 10）

### 日志
- LOG_LEVEL（debug/info/warn/error，默认 info）
- LOG_PATH（默认 /logs）

### 其他
- NODE_ENV（development/test/production）
- PORT（默认 8902）
- CORS_ORIGIN（允许的跨域域名，多个用逗号分隔）

---

## 部署方式

### 后端（ucar-api）
- 使用 Docker 部署：
```bash
docker build -t ucar-api .
docker run -d -p 8902:8902 --env-file .env ucar-api
```

### 管理后台前端（ucar-admin-web）
- 构建：
```bash
npm run build
```

- 部署到 Nginx：
```nginx
server {
  listen 80;
  server_name admin.example.com;
  root /var/www/ucar-admin-web/dist;
  index index.html;
}
```

### 微信小程序（ucar-miniapp）
- 导入微信开发者工具
- 配置 AppID 与服务器地址
- 上传审核并发布

## 日志规范

### 日志格式
统一使用 JSON 格式，包含以下字段：
```json
{
  "timestamp": "2026-04-22T10:30:00.000Z",
  "level": "info",
  "trace_id": "uuid",
  "module": "lead",
  "message": "新增线索成功",
  "user_id": "admin001",
  "ip": "192.168.1.1",
  "duration_ms": 45,
  "extra": {}
}
```

### 日志级别
- `debug`：开发调试信息，生产环境不输出
- `info`：正常业务日志（接口请求、状态变更）
- `warn`：警告信息（频率限制触发、非关键异常）
- `error`：错误信息（数据库连接失败、接口异常）

### 日志切割
- 按日期切割：`<LOG_PATH>/app-YYYY-MM-DD.log`
- 单文件最大 100MB，保留最近 30 天
- 错误日志单独存储：`<LOG_PATH>/error-YYYY-MM-DD.log`

## 监控指标

### 系统监控（Prometheus）
- CPU 使用率、内存使用率、磁盘使用率
- Node.js 事件循环延迟、GC 频率
- MySQL 连接数、慢查询数量

### 业务监控
- 接口 QPS、P50/P95/P99 响应时间
- 错误率（按接口维度统计）
- 登录成功率、微信登录成功率
- 线索转化率 = 成交数 / 线索总数
- 结算完成率 = 已结算数 / 结算总数

### 告警规则
- 接口 P99 响应时间 > 1000ms，持续 5 分钟
- 错误率 > 5%，持续 3 分钟
- 数据库连接数 > 80% 最大连接数
- 磁盘使用率 > 85%
- 登录接口失败率 > 20%，持续 5 分钟

## 安全配置
- 所有接口必须通过 HTTPS
- 配置跨域（CORS），仅允许指定域名访问
- 数据库账号需最小权限原则

## RSA密钥对配置

### 生成方法
```bash
# 在ucar-api目录下执行
openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -pubout -out public.pem
```

### 文件放置位置
- `public.pem`：放置在ucar-api根目录，用于密码加密
- `private.pem`：妥善保管，用于解密查看用户真实密码（仅管理员使用）

### 作用
- **public.pem**：系统使用公钥对用户密码进行RSA加密，存储到数据库的`password_secure`字段
- **private.pem**：管理员使用私钥解密`password_secure`字段，查看用户真实密码

### 注意事项
- 私钥文件必须妥善保管，避免泄露
- 每次部署时确保公钥文件存在于正确位置
- 不要将私钥文件提交到版本控制系统

## 数据库备份与恢复

### 备份策略
- 每日凌晨 2:00 自动全量备份
- 备份文件命名：`backup-YYYY-MM-DD-HH-mm-ss.sql`
- 存储路径：`/backup`
- 保留策略：保留最近 7 天全量备份
- 备份文件大小超过 1GB 时进行压缩存储（.gz）

### 恢复方式
```bash
# 恢复全量备份
mysql -u user -p db_name < backup-2026-04-22-02-00-00.sql

# 恢复压缩备份
gunzip < backup-2026-04-22-02-00-00.sql.gz | mysql -u user -p db_name
```

### 回滚方案
- 部署前创建数据库快照
- 应用升级失败时，回滚到上一个版本的 Docker 镜像
- 数据库变更使用迁移脚本（migration），支持回滚

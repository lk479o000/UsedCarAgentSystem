# 小程序 Lucide 图标手动添加指南

## 背景

本项目使用 `unocss` + `@unocss/preset-icons` + `@iconify-json/lucide` 自动生成图标样式。但 `@iconify-json/lucide` 依赖的版本可能滞后于 Lucide 官方图标库，导致部分新图标（如 `handshake`、`ban`）无法通过 `npm run unocss` 自动生成。

**核心原则**：
- `npm run unocss` 只会生成 `uno.config.js` 中 `safelist` 列出的图标 + 在 `.wxml`/`.js` 文件中被扫描到的图标
- 如果图标在 `@iconify-json/lucide` 依赖版本中不存在，即使 safelist 中有也不会生成
- **手动写入 `unocss.wxss` 的图标会被 `npm run unocss` 覆盖丢失**

---

## 两种添加图标的方式

### 方式一：自动方式（推荐，适用于已有图标）

1. 在 `uno.config.js` 的 `safelist` 中添加图标类名
2. 在 `.wxml` 或 `.js` 中使用该图标类名
3. 运行 `npm run unocss`
4. 检查 `unocss.wxss` 中是否生成了对应的 `.i-lucide-xxx` 规则

### 方式二：手动方式（适用于新图标或自动方式失效）

当自动方式无法生成图标时，需要手动将 SVG 数据写入 `unocss.wxss`，并确保该图标不会被 `unocss` 覆盖。

---

## 手动添加图标的完整流程

### 步骤 1：确认图标在 `@iconify-json/lucide` 中是否存在

```bash
cd ucar-miniapp
node -e "console.log(Object.keys(require('@iconify-json/lucide').icons).filter(k => k.includes('handshake')))"
```

如果返回空数组，说明当前依赖版本没有该图标，需要手动添加。

### 步骤 2：从 Lucide 官网获取 SVG 路径数据

**推荐方法：通过 GitHub API 获取官方 SVG 源码**

访问 `https://api.github.com/repos/lucide-icons/lucide/contents/icons/{图标名}.svg`

例如获取 `ban` 图标：
```bash
curl https://api.github.com/repos/lucide-icons/lucide/contents/icons/ban.svg
```

返回的 JSON 中 `content` 字段是 Base64 编码的 SVG 内容，解码后得到：

```svg
<svg
  xmlns="http://www.w3.org/2000/svg"
  width="24"
  height="24"
  viewBox="0 0 24 24"
  fill="none"
  stroke="currentColor"
  stroke-width="2"
  stroke-linecap="round"
  stroke-linejoin="round"
>
  <circle cx="12" cy="12" r="10" />
  <path d="M4.929 4.929 19.07 19.071" />
</svg>
```

### 步骤 3：将 SVG 转换为 unocss 格式

转换规则：
1. 移除 `<svg>` 标签上的 `width`、`height`、`fill` 属性（保留 `viewBox` 和 `xmlns`）
2. 将 `stroke="currentColor"`、`stroke-width="2"`、`stroke-linecap="round"`、`stroke-linejoin="round"` 写入每个子元素，或包裹在 `<g>` 组上
3. 将 SVG 内容 URL 编码后嵌入 CSS

**关键注意事项**：
- **`<circle>`、`<rect>`、`<line>`、`<polyline>` 等几何元素必须显式设置 `stroke="currentColor"`**，否则无法显示
- **多路径图标**：如果 SVG 有多个 `<path>`，每个都要带 `stroke` 属性，或者用 `<g>` 统一设置
- **路径中的空格**：`d` 属性值中的坐标对用空格分隔，如 `M4.929 4.929 L19.07 19.071`

### 步骤 4：写入 `unocss.wxss`

在 `/* layer: icons */` 区域，按照字母顺序插入新的图标规则：

```css
.i-lucide-ban{--un-icon:url("data:image/svg+xml;utf8,%3Csvg viewBox='0 0 24 24' display='inline-block' vertical-align='middle' width='1em' height='1em' xmlns='http://www.w3.org/2000/svg' %3E%3Cg fill='none' stroke='currentColor' stroke-linecap='round' stroke-linejoin='round' stroke-width='2'%3E%3Ccircle cx='12' cy='12' r='10'/%3E%3Cpath d='m4.9 4.9 14.2 14.2'/%3E%3C/g%3E%3C/svg%3E");-webkit-mask:var(--un-icon) no-repeat;mask:var(--un-icon) no-repeat;-webkit-mask-size:100% 100%;mask-size:100% 100%;background-color:currentColor;color:inherit;display:inline-block;vertical-align:middle;width:1em;height:1em;}
```

**格式模板**：
```css
.i-lucide-{图标名}{--un-icon:url("data:image/svg+xml;utf8,{URL编码后的SVG}");-webkit-mask:var(--un-icon) no-repeat;mask:var(--un-icon) no-repeat;-webkit-mask-size:100% 100%;mask-size:100% 100%;background-color:currentColor;color:inherit;display:inline-block;vertical-align:middle;width:1em;height:1em;}
```

### 步骤 5：防止被 `npm run unocss` 覆盖

**方法 A（推荐）：将图标加入 `safelist` + 在代码中使用**

在 `uno.config.js` 的 `safelist` 中添加图标名，并在 `.wxml`/`.js` 中使用。如果 `@iconify-json/lucide` 版本已包含该图标，unocss 会自动生成；如果不包含，手动写入的规则会保留（因为 unocss 扫描不到该图标的源码，不会生成新规则覆盖）。

**方法 B：备份手动规则**

运行 `npm run unocss` 后，检查手动添加的图标是否被覆盖。如果被覆盖，需要重新手动写入。

### 步骤 6：在代码中使用图标

```xml
<!-- WXML 示例 -->
<view class="i-lucide-handshake"></view>
<view class="i-lucide-ban"></view>
```

```js
// JS 示例（动态设置图标）
icon: 'i-lucide-handshake'
```

### 步骤 7：验证图标显示

1. 运行 `npm run unocss`
2. 在微信开发者工具中刷新预览
3. 检查图标是否正常显示
4. 如果图标不显示，检查：
   - `unocss.wxss` 中是否存在该图标的 CSS 规则
   - SVG 路径数据是否正确（特别是 `stroke` 属性）
   - 元素是否正确应用了图标类名

## 常见问题排查

### 问题 1：`npm run unocss` 后手动添加的图标丢失

**原因**：unocss 重新生成了整个 `unocss.wxss` 文件，只包含它能扫描到的图标。

**解决**：
- 确保图标类名在 `uno.config.js` 的 `safelist` 中
- 确保图标在 `.wxml` 或 `.js` 文件中被使用
- 如果 `@iconify-json/lucide` 版本不支持，需要每次运行 `npm run unocss` 后重新手动写入

### 问题 2：图标显示为空白或只有部分路径

**原因**：SVG 中的几何元素（`<circle>`、`<rect>`、`<line>` 等）缺少 `stroke` 属性。

**解决**：将所有子元素包裹在 `<g>` 组中，统一设置 `stroke="currentColor"`：

```svg
<!-- 错误 -->
<svg ...>
  <circle cx="12" cy="12" r="10" />
  <path d="..." />
</svg>

<!-- 正确 -->
<svg ...>
  <g fill="none" stroke="currentColor" stroke-linecap="round" stroke-linejoin="round" stroke-width="2">
    <circle cx="12" cy="12" r="10" />
    <path d="..." />
  </g>
</svg>
```

### 问题 3：图标颜色不跟随父元素

**原因**：SVG 中硬编码了颜色，或未使用 `currentColor`。

**解决**：确保 SVG 中使用 `stroke="currentColor"` 或 `fill="currentColor"`，不要写死颜色值。

# 数据库设计

## 表：c_user（用户）
- id (INT, PK, AUTO_INCREMENT)
- username (VARCHAR(50), UNIQUE)  # 管理员账号
- password (VARCHAR(100))  # 系统登录密码（使用bcrypt哈希存储）
- password_secure (VARCHAR(500))  # 安全密码（使用RSA公钥加密存储）
- name (VARCHAR(50), NOT NULL)  # 真实姓名
- nickname (VARCHAR(50))  # 昵称
- phone (VARCHAR(20), UNIQUE)  # 经纪人手机号
- open_id (VARCHAR(100), UNIQUE)  # 微信用户唯一标识
- role (TINYINT, NOT NULL, DEFAULT 1)  # 0=管理员, 1=普通用户（经纪人）
- status (TINYINT, NOT NULL, DEFAULT 1)  # 1=启用, 0=禁用
- remark (TEXT)  # 备注
- created_at (DATETIME, DEFAULT CURRENT_TIMESTAMP)
- updated_at (DATETIME, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)

索引：

- username（唯一索引）
- phone（唯一索引）
- open\_id（唯一索引）

***

## 表：c\_lead（线索）

- id (INT, PK, AUTO\_INCREMENT)
- customer\_name (VARCHAR(50), NOT NULL)
- customer\_phone (VARCHAR(20), NOT NULL)
- car\_brand (VARCHAR(50), NOT NULL)
- car\_model (VARCHAR(50), NOT NULL)
- notes (TEXT)
- user_id (INT, FK → c_user.id, NOT NULL)
- status (TINYINT, NOT NULL, DEFAULT 0)  # 0=待跟进, 1=跟进中, 2=已看车, 3=已报价, 4=已成交, 5=已失败
- fail_reason (VARCHAR(100))
- created_at (DATETIME, DEFAULT CURRENT_TIMESTAMP)
- updated_at (DATETIME, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)
- last_followup_at (DATETIME)

索引：

- user\_id（外键索引）
- status（普通索引）
- customer\_phone（普通索引）

***

## 表：c\_settlement（结算）

- id (INT, PK, AUTO\_INCREMENT)
- lead\_id (INT, FK → c\_lead.id, NOT NULL)
- profit (DECIMAL(10,2), NOT NULL)
- agent\_share (DECIMAL(10,2), NOT NULL)
- status (TINYINT, NOT NULL, DEFAULT 0)  # 0=待结算, 1=已结算
- remark (TEXT)
- settled_at (DATETIME)
- user_id (INT, FK → c_user.id)  # 结算操作人（管理员）

索引：

- lead\_id（外键索引）
- status（普通索引）

***

***

## 表：c\_operation\_log（操作日志）

- id (INT, PK, AUTO\_INCREMENT)
- user_id (INT)  # 操作用户ID（c_user或c_admin）
- user_type (TINYINT)  # 用户类型（0=管理员, 1=经纪人）
- operation_type (TINYINT, NOT NULL)  # 操作类型（0=登录, 1=新增, 2=修改, 3=删除, 4=其他）
- operation_target (TINYINT, NOT NULL)  # 操作目标（0=线索, 1=结算, 2=用户, 3=其他）
- target_id (INT)  # 目标ID
- operation_content (TEXT)  # 操作内容
- ip_address (VARBINARY(16))  # 操作IP地址（使用VARBINARY存储，更节省空间）
- created\_at (DATETIME, DEFAULT CURRENT\_TIMESTAMP)

索引：

- user\_id（普通索引）
- created\_at（普通索引）

***

## 表：c_dict（码表/字典表）

- id (INT, PK, AUTO_INCREMENT)
- dict_type (VARCHAR(50), NOT NULL)  # 字典类型（如user_role, user_status, lead_status等）
- dict_code (TINYINT, NOT NULL)  # 字典编码
- dict_name (VARCHAR(50), NOT NULL)  # 字典名称
- dict_desc (VARCHAR(100))  # 字典描述
- sort_order (TINYINT, NOT NULL, DEFAULT 0)  # 排序顺序
- status (TINYINT, NOT NULL, DEFAULT 1)  # 1=启用, 0=禁用
- created_at (DATETIME, DEFAULT CURRENT_TIMESTAMP)
- updated_at (DATETIME, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)

索引：

- dict_type, dict_code（唯一索引）

***

## 表关系说明

- 一个用户（c\_user）可以有多个线索（c\_lead），但只有角色为经纪人（role=1）的用户才能拥有线索。
- 一个线索（c\_lead）对应一个结算记录（c\_settlement），仅当状态为已成交时才生成。
- 结算记录（c\_settlement）由管理员（role=0）操作，关联到线索（c\_lead）和操作用户（c\_user）。
- 所有用户的操作都会记录到操作日志表（c\_operation\_log）中。

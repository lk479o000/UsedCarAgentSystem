# 数据库设计

## 表：c_user（用户）

- id (BIGINT, PK, AUTO_INCREMENT)  # 用户ID（主键）
- open_id (VARCHAR(100), NULL, UNIQUE)  # 微信用户唯一标识
- userid (VARCHAR(15), NOT NULL, UNIQUE)  # 用户账号
- username (VARCHAR(50), NOT NULL)  # 真实姓名
- password (VARCHAR(100), NOT NULL)  # 系统登录密码（使用bcrypt哈希存储）
- password_secure (VARCHAR(500), NOT NULL)  # 安全密码（使用RSA公钥加密存储）
- nickname (VARCHAR(50), NULL)  # 昵称
- phone (VARCHAR(20), NOT NULL, UNIQUE)  # 手机号
- headimgurl (VARCHAR(255), NULL)  # 用户头像URL
- wechat_bound (TINYINT, NOT NULL, DEFAULT 0)  # 微信是否绑定（ 0=未绑定, 1=已绑定 ）
- role (TINYINT, NOT NULL, DEFAULT 1)  # 用户角色（ 0=管理员, 1=普通用户（经纪人） ）
- status (TINYINT, NOT NULL, DEFAULT 1)  # 用户状态（ 1=启用, 0=禁用 ）
- remark (TEXT, NULL)  # 备注
- created_at (DATETIME, NOT NULL, DEFAULT CURRENT_TIMESTAMP)  # 创建时间
- updated_at (DATETIME, NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)  # 更新时间（自动更新）
- is_deleted (TINYINT, NOT NULL, DEFAULT 0)  # 是否删除（ 1=已删除, 0=未删除 ）
- deleted_at (DATETIME, NULL)  # 删除时间

**业务规则：**

- 管理员（role=0）可使用账号密码登录，也可绑定微信后使用微信登录
- 经纪人（role=1）可使用账号密码登录，也可绑定微信后使用微信登录
- 微信登录前必须绑定手机号
- 禁用状态（status=0）的用户无法登录系统
- 删除用户时，其关联的线索和结算记录保留，但不可再归属新线索

**索引：**

- userid（唯一索引）
- phone（唯一索引）
- open_id（唯一索引）
- role（普通索引）
- status（普通索引）
- is_deleted（普通索引）

## 表：c_lead（线索）

- id (BIGINT, PK, AUTO_INCREMENT)  # 线索ID（主键）
- customer_type (TINYINT, NOT NULL, DEFAULT 0)  # 客户类型（ 0=买家, 1=卖家 ）
- customer_name (VARCHAR(50), NOT NULL)  # 客户姓名
- customer_phone (VARCHAR(20), NOT NULL)  # 客户手机号
- car_brand (VARCHAR(50))  # 车辆品牌
- car_model (VARCHAR(50), NOT NULL)  # 车辆型号
- province_id (BIGINT, NULL)  # 所属省份ID
- city_id (BIGINT, NULL)  # 所属城市ID
- district_id (BIGINT, NULL)  # 所属区县ID
- car_price (BIGINT, NULL, DEFAULT 0)  # 预估金额
- car_actual_price (BIGINT, NULL, DEFAULT 0)  # 实际成交金额
- notes (VARCHAR(500), NULL)  # 线索备注
- user_id (VARCHAR(15), NOT NULL, FK → c_user.userid, NOT NULL)  # 关联的经纪人ID（外键）
- status (TINYINT, NOT NULL, DEFAULT 0)  # 线索状态（ 0=待跟进, 1=跟进中, 2=已看车, 3=已报价, 4=已成交, 5=已失败 ）
- fail_reason (VARCHAR(500), NULL)  # 失败原因（仅在状态为5时填写）
- created_at (DATETIME, NOT NULL, DEFAULT CURRENT_TIMESTAMP)  # 创建时间
- updated_at (DATETIME, NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)  # 更新时间（自动更新）
- last_followup_at (DATETIME, NULL)  # 最后跟进时间
- is_deleted (TINYINT, NOT NULL, DEFAULT 0)  # 是否删除（ 1=已删除, 0=未删除 ）
- deleted_at (DATETIME, NULL)  # 删除时间

**业务规则：**

- 线索状态流转需遵循状态机规则：待跟进 → 跟进中 → 已看车 → 已报价 → 已成交/已失败
- 状态为已成交后不可再修改，系统自动生成结算记录
- 状态为已失败时必须填写失败原因
- 已成交和已失败的线索不可删除

**索引：**

- user_id（外键索引）
- province_id（普通索引）
- city_id（普通索引）
- district_id（普通索引）
- status（普通索引）
- customer_phone（普通索引）
- created_at（普通索引，倒序排序）
- is_deleted（普通索引）

## 表：c_settlement（结算）

- id (BIGINT, PK, AUTO_INCREMENT)  # 结算ID（主键）
- lead_id (BIGINT, FK → c_lead.id, NOT NULL)  # 关联的线索ID（外键）
- profit (BIGINT, NOT NULL, DEFAULT 0)  # 利润金额（单位：元）
- agent_share (BIGINT, NOT NULL, DEFAULT 0)  # 经纪人分享金额（单位：元，默认利润金额的50%，不可超过利润金额）
- status (TINYINT, NOT NULL, DEFAULT 0)  # 结算状态（ 0=待结算, 1=已结算 ）
- remark (VARCHAR(500), NULL)  # 结算备注
- settled_at (DATETIME, NULL)  # 结算时间
- user_id (VARCHAR(15), NOT NULL, FK → c_user.userid, NULL)  # 结算操作人（管理员）
- is_deleted (TINYINT, NOT NULL, DEFAULT 0)  # 是否删除（ 1=已删除, 0=未删除 ）
- deleted_at (DATETIME, NULL)  # 删除时间

**业务规则：**

- 仅当线索状态为已成交时才可创建结算记录
- agent_share 必须小于等于 profit
- 已结算的记录不可修改金额，仅允许修改备注
- 结算记录删除需同步检查线索状态

**索引：**

- lead_id（外键索引）
- status（普通索引）
- user_id（外键索引）
- is_deleted（普通索引）

## 表：c_operation_log（操作日志）

- id (BIGINT, PK, AUTO_INCREMENT)
- user_id (VARCHAR(15), NOT NULL), NOT NULL)  # 操作用户ID（c_user）
- user_type (TINYINT, NOT NULL)  # 用户类型（0=管理员, 1=经纪人）
- operation_type (TINYINT, NOT NULL)  # 操作类型（0=登录, 1=新增, 2=修改, 3=删除, 4=其他）
- operation_target (TINYINT, NOT NULL)  # 操作目标（0=线索, 1=结算, 2=用户, 3=其他）
- target_id (BIGINT, NULL)  # 目标ID
- operation_content (TEXT, NULL)  # 操作内容
- ip_address (VARBINARY(16), NULL)  # 操作IP地址（使用VARBINARY存储，更节省空间）
- created_at (DATETIME, NOT NULL, DEFAULT CURRENT_TIMESTAMP)  # 创建时间
- updated_at (DATETIME, NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)  # 更新时间（自动更新）
- is_deleted (TINYINT, NOT NULL, DEFAULT 0)  # 是否删除（ 1=已删除, 0=未删除 ）
- deleted_at (DATETIME, NULL)  # 删除时间

**索引：**

- user_id（普通索引）
- created_at（普通索引，倒序排序）
- (user_id, created_at)（联合索引）
- is_deleted（普通索引）

- id (BIGINT, PK, AUTO_INCREMENT)  # 字典ID（主键）

- updated_at (DATETIME, NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)  # 更新时间（自动更新）
- is_deleted (TINYINT, NOT NULL, DEFAULT 0)  # 是否删除（ 1=已删除, 0=未删除 ）
- deleted_at (DATETIME, NULL)  # 删除时间

**索引：**

- dict_type, dict_code（唯一索引）
- status（普通索引）
- is_deleted（普通索引）

## 表：c_lead_import（线索导入记录表）

- id (BIGINT, PK, AUTO_INCREMENT)  # 导入记录ID（主键）
- customer_name (VARCHAR(50))  # 客户名称
- customer_phone (VARCHAR(20))  # 电话号码（可能是手机号或微信号）
- referrer (VARCHAR(50), NULL)  # 介绍人
- expected_price (VARCHAR(20), NULL)  # 心里价位（万），原始文本，可能含范围
- wechat_added (TINYINT, NULL)  # 是否添加微信（0=未添加, 1=个人微信, 2=企业微信）
- car_brand (VARCHAR(50), NULL)  # 车型（品牌+型号，如"奔驰C260"）
- vin (VARCHAR(50), NULL)  # 车架号（VIN码）
- mileage (VARCHAR(20), NULL)  # 公里数（万），原始文本
- is_dealt (TINYINT, NULL)  # 是否成交（0=否, 1=是）
- estimated_amount (VARCHAR(20), NULL)  # 预估金额，原始文本
- deal_amount (VARCHAR(20), NULL)  # 成交金额（万），原始文本
- is_sold (TINYINT, NULL)  # 是否已售（0=否, 1=是）
- buyer_name (VARCHAR(50), NULL)  # 回购人员
- car_address (VARCHAR(200), NULL)  # 车辆地址
- import_date (DATE, NULL)  # 原始日期（Excel中的日期）
- import_batch (VARCHAR(50), NOT NULL)  # 导入批次号（如"20251126"）
- import_status (TINYINT, NOT NULL, DEFAULT 0)  # 导入状态（0=未处理, 1=已转为线索, 2=已忽略）
- lead_id (BIGINT, NULL)  # 关联的线索ID（转为线索后填充）
- remark (VARCHAR(500), NULL)  # 处理备注
- created_at (DATETIME, NOT NULL, DEFAULT CURRENT_TIMESTAMP)  # 创建时间
- updated_at (DATETIME, NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)  # 更新时间
- is_deleted (TINYINT, NOT NULL, DEFAULT 0)  # 是否删除（ 1=已删除, 0=未删除 ）
- deleted_at (DATETIME, NULL)  # 删除时间

**业务规则：**

- 该表用于存储从 Excel 导入的原始客户跟进数据
- 导入时保留原始文本格式，不做数据清洗
- 管理员可在后台查看导入记录，选择转为正式线索或忽略
- 转为线索时，需手动指定归属经纪人，并补充必要字段
- 同一客户多次导入时，根据手机号/客户名称去重提示

**索引：**

- import_batch（普通索引）
- customer_phone（普通索引）
- import_status（普通索引）
- (customer_name, customer_phone)（联合索引，用于去重）

***

## 表关系说明

- 一个用户（c_user）可以有多个线索（c_lead），但只有角色为经纪人（role=1）的用户才能拥有线索，仅管理员能创建线索。
- 一个线索（c_lead）对应一个结算记录（c_settlement），仅当状态为已成交时才生成, 已成交后不能修改。
- 结算记录（c_settlement）由管理员（role=0）操作，关联到线索（c_lead）和操作用户（c_user）。
- 所有用户的操作都会记录到操作日志表（c_operation_log）中。
- 导入记录（c_lead_import）可由管理员转为线索（c_lead），转为线索后更新 import_status 和 lead_id。

# 小程序功能规划方案

## 一、PC端功能全景分析

### PC端核心模块

| 页面 | 核心功能 |
|------|--------|
| **登录页** | 账号密码登录、验证码、记住密码 |
| **首页/仪表盘** | 统计数据展示（总线索、已成交、已失败、待结算金额、已结算金额）、图表占位 |
| **线索管理** | 列表、搜索、新增、编辑、删除、更新状态（6种状态流转）、跟进记录（增删改查）、结算/重新结算、导出Excel |
| **结算管理** | 列表、搜索、新增、编辑、更新状态（待结算/已结算）、推送提醒、导出Excel |
| **经纪人管理** | 列表、搜索、新增、编辑、启用/禁用、重置密码（仅管理员可见） |
| **个人设置** | 修改密码（含密码强度检测） |

### PC端API接口

| 模块 | 接口 | 说明 |
|------|------|------|
| **线索** | `GET /lead` | 管理员-线索列表 |
| | `POST /lead` | 新增线索 |
| | `PUT /lead/:id` | 更新线索 |
| | `DELETE /lead/:id` | 删除线索 |
| | `POST /lead/:id/followup` | 新增跟进记录 |
| | `GET /lead/:id/followup` | 获取跟进记录 |
| | `PUT /lead/followup/:id` | 更新跟进记录 |
| | `DELETE /lead/followup/:id` | 删除跟进记录 |
| **结算** | `GET /settlement` | 管理员-结算列表 |
| | `POST /settlement` | 新增结算 |
| | `PUT /settlement/:id` | 更新结算 |
| | `GET /settlement/lead/:leadId` | 按线索ID获取结算 |
| **用户** | `GET /user/lead` | 经纪人-线索列表 |
| | `GET /user/settlement` | 经纪人-结算列表 |
| | `GET /user/overview` | 统计概览 |

## 二、PC端角色权限详细分析

### 角色定义

| 角色 | role值 | 判断方式 |
|------|--------|----------|
| 管理员 (Admin) | 0 | `userStore.isAdmin === true` |
| 经纪人 (Agent) | 非0 | `userStore.isAdmin === false` |

### 经纪人权限总结

> **核心结论：经纪人在PC端是只读角色，只能查看属于自己的数据，不能进行任何新增、编辑、删除、导出、状态变更或结算操作。但经纪人可以查看跟进记录。**

### 线索管理 - 角色权限对比

| 维度 | 管理员 | 经纪人 |
|------|--------|--------|
| **数据范围** | 全部线索 (`GET /lead`) | 仅自己的线索 (`GET /user/lead`) |
| **可见表格列** | 客户姓名、客户电话、客户类型、车辆品牌、车辆型号、**归属经纪人**、状态、创建时间、**操作** | 客户姓名、客户电话、客户类型、车辆品牌、车辆型号、状态、创建时间 |
| **隐藏字段** | 无 | 归属经纪人、操作列 |
| **详情弹窗字段** | 客户姓名、客户电话、客户类型、车辆品牌、车辆型号、状态、**归属经纪人**、备注、创建时间、**跟进记录（增删改查）** | 客户姓名、客户电话、客户类型、车辆品牌、车辆型号、状态、备注、创建时间、**跟进记录（仅查看）** |
| **可执行操作** | 导出Excel、新增、更新状态、编辑、跟进记录（增删改查）、结算、重新结算、删除 | **仅查看**（搜索、分页、查看详情、查看跟进记录） |
| **不可执行操作** | 无 | 导出Excel、新增、更新状态、编辑、跟进记录（增删改）、结算、重新结算、删除 |

### 结算管理 - 角色权限对比

| 维度 | 管理员 | 经纪人 |
|------|--------|--------|
| **数据范围** | 全部结算 (`GET /settlement`) | 仅自己的结算 (`GET /user/settlement`) |
| **可见表格列** | 客户姓名、车辆品牌、车辆型号、利润金额、经纪人分成、状态、结算时间、**操作** | 客户姓名、车辆品牌、车辆型号、利润金额、经纪人分成、状态、结算时间 |
| **隐藏字段** | 无 | 操作列 |
| **详情弹窗字段** | 客户姓名、车辆品牌、车辆型号、利润金额、经纪人分成、状态、结算时间、备注、创建时间 | 客户姓名、车辆品牌、车辆型号、利润金额、经纪人分成、状态、结算时间、备注、创建时间 |
| **可执行操作** | 导出Excel、新增结算、更新状态、推送提醒、编辑 | **仅查看**（搜索、分页、查看详情） |
| **不可执行操作** | 无 | 导出Excel、新增结算、更新状态、推送提醒、编辑 |

### 仪表盘 - 角色权限对比

| 维度 | 管理员 | 经纪人 |
|------|--------|--------|
| **可见内容** | 统计卡片 + 图表占位 | 统计卡片 + 图表占位 |
| **数据来源** | `GET /user/overview` | `GET /user/overview`（后端按角色返回不同数据） |

### 侧边栏菜单 - 角色权限对比

| 菜单项 | 管理员 | 经纪人 |
|--------|--------|--------|
| 首页 | ✅ | ✅ |
| 线索管理 | ✅ | ✅ |
| 结算管理 | ✅ | ✅ |
| 经纪人管理 | ✅ | ❌ |

## 三、小程序端页面功能规划

### 交互方式说明

| 场景 | 交互方式 | 说明 |
|------|----------|------|
| 列表→详情 | **点击卡片进入独立详情页** | 不采用行内展开，因为手机屏幕空间有限，独立页面能展示更完整的信息和操作按钮 |
| 列表→新增 | **点击右上角"+"按钮进入表单页** | tabBar页面使用页面内悬浮按钮，非tabBar页面使用导航栏按钮 |
| 详情→编辑 | **详情页底部操作栏"编辑"按钮** | 进入编辑表单页 |
| 详情→删除 | **详情页底部操作栏"删除"按钮** | 弹出确认弹窗 |
| 详情→更新状态 | **详情页底部操作栏"更新状态"按钮** | 弹出底部ActionSheet选择新状态 |
| 详情→跟进记录 | **详情页内嵌跟进记录列表** | 不单独页面，在详情页下半部分展示 |
| 详情→结算 | **详情页底部操作栏"结算"按钮** | 进入结算表单页 |
| 表单提交 | **表单页底部"确定"按钮** | 提交后返回上一页并刷新数据 |

### 页面结构总览

```
ucar-miniapp/
├── pages/
│   ├── login/              # 登录（现有）
│   ├── index/              # 首页（增强）
│   ├── leads/
│   │   ├── leads           # 线索列表（增强）
│   │   ├── detail          # 线索详情（新增）
│   │   └── form            # 线索表单（新增）
│   ├── followups/
│   │   └── form            # 跟进记录表单（新增）
│   ├── settlements/
│   │   ├── settlements     # 结算列表（增强）
│   │   ├── detail          # 结算详情（新增）
│   │   └── form            # 结算表单（新增）
│   ├── profile/            # 个人中心（增强）
│   ├── bind-phone/         # 绑定手机号（现有）
│   ├── agreement/          # 用户协议（现有）
│   └── privacy/            # 隐私政策（现有）
```

## 四、各页面详细设计

### 1. 首页 (`pages/index/index`)

**角色权限：** 管理员和经纪人共用，无差异

| 车辆信息 | `carBrand + carModel` | 灰色小字 |

### 2. 线索列表页 (`pages/leads/leads`)

**导航栏标题：** 线索管理  
**角色权限：** 管理员可新增；经纪人纯查看

#### 顶部筛选栏

| 筛选项 | 选项 | 说明 |
|--------|------|------|
| 状态 | 全部、待跟进、跟进中、已看车、已报价、已成交、已失败 | 横向滑动标签栏 |
| 搜索 | 客户姓名 | 搜索输入框（筛选栏下方） |

> 手机端7个状态标签无法一行展示，采用**横向滚动**方式

#### 新增按钮

| 角色 | 是否显示 | 位置 |
|------|----------|------|
| 管理员 | ✅ | 页面右下角悬浮"+"按钮 |
| 经纪人 | ❌ | 无 |

#### 线索卡片（列表项）

每张卡片展示以下字段，采用**纵向堆叠**布局：

```
┌─────────────────────────────────┐
│  张三          [已成交]          │  ← 第一行：客户姓名 + 状态标签
│  138****1234                    │  ← 第二行：客户电话
│  ─────────────────────────────  │  ← 分割线
│  [宝马]  5系 2024款             │  ← 第三行：车辆品牌 + 型号
│  2024-01-15 10:30              │  ← 第四行：创建时间（灰色小字）
└─────────────────────────────────┘
```

| 字段 | 管理员可见 | 经纪人可见 | 展示方式 |
|------|-----------|-----------|----------|
| 客户姓名 | ✅ | ✅ | 加粗，左侧 |
| 状态标签 | ✅ | ✅ | 右侧彩色标签 |
| 客户电话 | ✅ | ✅ | 灰色文字 |
| 车辆品牌 | ✅ | ✅ | 灰色背景小标签 |
| 车辆型号 | ✅ | ✅ | 普通文字 |
| 创建时间 | ✅ | ✅ | 底部灰色小字 |
| 归属经纪人 | ✅ | ❌ | 管理员额外显示一行 |

> **手机端适配说明：**  
> - 客户电话考虑隐私，经纪人视角下中间4位用`****`替代（如138\*\*\*1234），管理员可看完整号码
> - 车辆品牌+型号合并为一行展示，节省纵向空间
> - 客户类型（买家/卖家）不在列表卡片中展示，放入详情页，避免卡片信息过载

#### 交互行为

| 操作 | 行为 |
|------|------|
| 点击卡片 | `wx.navigateTo('/pages/leads/detail?id=xxx')` 进入详情页 |
| 下拉刷新 | 重置页码，重新加载数据 |
| 上拉加载 | 加载下一页数据 |
| 点击悬浮"+" | `wx.navigateTo('/pages/leads/form')` 进入新增表单（仅管理员） |

### 3. 线索详情页 (`pages/leads/detail`)

**导航栏标题：** 线索详情  
**角色权限：** 管理员可操作；经纪人纯查看

#### 基本信息区域

| 字段 | 管理员可见 | 经纪人可见 | 展示方式 |
|------|-----------|-----------|----------|
| 客户姓名 | ✅ | ✅ | 左侧标签 + 右侧值 |
| 客户电话 | ✅ | ✅（中间4位\*\*\*\*） | 左侧标签 + 右侧值 |
| 客户类型 | ✅ | ✅ | 左侧标签 + 右侧值（买家/卖家） |
| 车辆品牌 | ✅ | ✅ | 左侧标签 + 右侧值 |
| 车辆型号 | ✅ | ✅ | 左侧标签 + 右侧值 |
| 状态 | ✅ | ✅ | 左侧标签 + 右侧彩色标签 |
| 归属经纪人 | ✅ | ❌ | 左侧标签 + 右侧值（仅管理员） |
| 备注 | ✅ | ✅ | 左侧标签 + 右侧值（多行文本） |
| 创建时间 | ✅ | ✅ | 左侧标签 + 右侧值 |

> **布局方式：** 每个字段占一行，左侧灰色标签（固定宽度80rpx），右侧黑色值

#### 跟进记录区域

| 角色 | 是否显示 | 可执行操作 |
|------|----------|-----------|
| 管理员 | ✅ | 新增、编辑、删除 |
| 经纪人 | ✅ | 仅查看（不显示新增/编辑/删除按钮） |

每条跟进记录卡片展示：

```
┌─────────────────────────────────┐
│  管理员        [编辑] [删除]      │  ← 操作人 + 操作按钮（仅管理员可见）
│  跟进时间：2024-01-15 10:30      │
│  跟进内容：客户已确认购车意向      │
│  跟进结果：准备报价               │
│  下次跟进：2024-01-20 14:00      │
└─────────────────────────────────┘
```

| 字段 | 管理员可见 | 经纪人可见 | 展示方式 |
|------|-----------|-----------|----------|
| 操作人 | ✅ | ✅ | 加粗文字 |
| 跟进时间 | ✅ | ✅ | 灰色小字 |
| 跟进内容 | ✅ | ✅ | 普通文字 |
| 跟进结果 | ✅ | ✅ | 普通文字（如有） |
| 下次跟进时间 | ✅ | ✅ | 灰色小字（如有） |
| 编辑按钮 | ✅ | ❌ | 文字按钮 |
| 删除按钮 | ✅ | ❌ | 红色文字按钮 |

> 跟进记录区域顶部的"新增跟进记录"按钮仅管理员可见

#### 底部操作栏

| 按钮 | 管理员可见 | 经纪人可见 | 说明 |
|------|-----------|-----------|------|
| 编辑 | ✅ | ❌ | 进入编辑表单页 |
| 更新状态 | ✅（状态非已成交时） | ❌ | 弹出底部ActionSheet选择新状态 |
| 结算 | ✅（状态为已成交且未结算时） | ❌ | 进入结算表单页 |
| 重新结算 | ✅（状态为已成交且已结算时） | ❌ | 进入结算表单页（预填数据） |
| 删除 | ✅ | ❌ | 弹出确认弹窗 |

> **经纪人底部操作栏：** 不显示任何操作按钮，详情页为纯查看模式  
> **管理员底部操作栏：** 根据线索状态动态显示可用操作按钮

#### 状态流转规则（与PC端一致）

```
待跟进(0) → 跟进中(1)
跟进中(1) → 已看车(2) / 已失败(5)
已看车(2) → 已报价(3) / 已失败(5)
已报价(3) → 已成交(4) / 已失败(5)
已成交(4) → 终态
已失败(5) → 终态
```

#### 更新状态弹窗（ActionSheet）

当选择"已成交"时，额外弹出输入框填写**成交价格**  
当选择"已失败"时，额外弹出输入框填写**失败原因**

## 六、手机端适配要点

### 列表页适配

| 问题 | 解决方案 |
|------|----------|
| 状态筛选标签过多（7个） | 横向滚动，当前选中项高亮 |
| 卡片信息密度高 | 分层展示：主信息（姓名+状态）→ 次信息（电话+车辆）→ 辅助信息（时间） |
| 客户电话隐私 | 经纪人视角中间4位用\*\*\*\*替代 |
| 客户类型不在列表展示 | 放入详情页，避免卡片过载 |
| 归属经纪人字段 | 仅管理员在列表卡片中额外显示一行 |

### 详情页适配

| 问题 | 解决方案 |
|------|----------|
| 字段较多 | 每个字段一行，左侧标签固定宽度，右侧值自适应 |
| 操作按钮多 | 底部固定操作栏，根据状态动态显示可用按钮 |
| 跟进记录列表 | 详情页内嵌，不单独页面，减少页面跳转层级 |
| 长文本（备注/跟进内容） | 自适应高度，不截断 |

### 表单页适配

| 问题 | 解决方案 |
|------|----------|
| 表单字段多 | 页面支持滚动，底部固定提交按钮 |
| 选择器交互 | 使用小程序原生Picker组件 |
| 线索选择 | 跳转线索列表页（选择模式），选中后返回 |
| 数字输入 | 使用数字键盘，限制小数位数 |

## 八、注意事项

1. **权限控制**：管理员和经纪人共用页面，通过角色判断显示/隐藏字段和操作按钮
2. **数据隔离**：经纪人只能查看自己的数据（后端接口已区分）
3. **隐私保护**：经纪人视角下客户电话中间4位脱敏
4. **表单验证**：与PC端保持一致的验证规则
5. **状态流转**：严格按照PC端的状态流转规则
6. **错误处理**：友好的错误提示，网络异常处理
7. **页面跳转**：使用`wx.navigateTo`保留返回栈，表单提交后使用`wx.navigateBack`返回并刷新

## 九、总结

本方案基于PC端现有功能，详细规划了小程序端的页面结构、字段展示、交互方式和权限控制，确保两端功能一致的同时，适配移动端交互特性。核心设计原则：

1. **功能对齐**：小程序端覆盖PC端所有业务功能（经纪人管理除外）
2. **权限一致**：管理员可操作，经纪人只读（可查看跟进记录但不可操作），与PC端权限逻辑保持一致
3. **移动适配**：卡片式列表、独立详情页、底部操作栏、横向滚动筛选等移动端交互模式
4. **隐私保护**：经纪人视角下敏感字段脱敏处理
5. **分阶段实施**：按优先级逐步开发，先保证核心查看功能，再补充操作功能

# 接口设计规范

## 通用说明
- 所有接口统一前缀：`/api/v1`
- 所有接口返回统一 JSON 格式：
```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```
- 鉴权方式：JWT，登录成功后返回 token，后续请求需在 Header 中携带：`Authorization: Bearer <token>`

### 分页参数规范
- 列表查询接口统一支持分页参数：
  - `page`：页码，从 1 开始，默认 1
  - `size`：每页数量，默认 20，最大 100
- 分页响应格式：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "pagination": {
      "page": 1,
      "size": 20,
      "total": 100,
      "total_pages": 5
    }
  }
}
```

### 接口频率限制
- 登录接口：同一 IP 每分钟最多 10 次请求
- 其他接口：同一用户每分钟最多 600 次请求
- 超出限制返回：`{ "code": 429, "message": "请求过于频繁，请稍后再试" }`

## 登录接口

### 获取验证码
- GET `/api/v1/auth/captcha`
- 功能：获取登录验证码
- 返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "captcha_id": "xxxx",
    "captcha_image": "data:image/png;base64,xxxx"
  }
}
```

### 管理员登录
- POST `/api/v1/auth/login`
- 请求参数：
```json
{
  "username": "admin",
  "password": "123456",
  "captcha": "1234",
  "captcha_id": "xxxx"
}
```
- 返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "xxxx",
    "user_info": {
      "id": 1,
      "username": "管理员",
      "role": 0,
      "status": 1
    }
  }
}
```

### 修改密码
- PUT `/api/v1/auth/password`
- 请求参数：
```json
{
  "old_password": "123456",
  "new_password": "654321",
  "new_password_confirm": "654321"
}
```
- 密码强度要求：
  - 密码长度至少8位
  - 必须包含字母和数字
  - 不能与原密码相同
  - 不能使用常见弱密码（如"123456"、"password"等）
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```
- 错误码：
  - `code = 1`：参数错误（包括密码强度不满足要求）
  - `code = 2`：原密码错误

### 设置安全密码
- PUT `/api/v1/auth/secure-password`
- 请求参数：
```json
{
  "secure_password": "your-secure-password"
}
```
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```

### 微信登录（管理员和经纪人通用）
- POST `/api/v1/auth/login/wechat`
- 功能：微信小程序 code2Session 登录，服务端调用微信接口换取 openid
- 请求参数：
```json
{
  "code": "wx_login_code",
  "nickname": "张三",
  "avatar_url": "https://example.com/avatar.jpg"
}
```
- 处理流程：
  1. 服务端使用 `code` + `WX_APPID` + `WX_SECRET` 请求微信 `jscode2session` 接口
  2. 微信返回 `openid` 和 `session_key`
  3. 服务端根据 `openid` 查询用户：
     - 若用户存在且状态正常：
       - 更新 `nickname`、`headimgurl`（如果提供）
       - 若已绑定手机号 → 直接返回 JWT token
       - 若未绑定手机号 → 返回 `need_bind_phone = true`
     - 若用户不存在：
       - 创建临时用户记录（`open_id`、`nickname`、`headimgurl`、`wechat_bound=1`、`status=1`、`role=1`）
       - 返回 `need_bind_phone = true`
  4. 记录登录日志到 `c_operation_log`
- 返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "xxxx",
    "user_info": {
      "id": 1,
      "nickname": "张三",
      "username": "",
      "phone": "",
      "role": 1,
      "status": 1,
      "wechat_bound": 1,
      "need_bind_phone": true
    }
  }
}
```
- 特殊情况：
  - 未绑定手机号的用户，`need_bind_phone` 为 `true`，需先调用绑定手机号接口
  - 禁用状态的用户无法登录
  - 临时用户（未绑定手机号）可登录系统，但无法查看线索数据

### 绑定手机号
- POST `/api/v1/auth/bind-phone`
- 功能：微信登录后绑定手机号，将微信账号与用户关联
- 请求参数：
```json
{
  "phone": "13800000000",
  "nickname": "张三",
  "avatar_url": "https://example.com/avatar.jpg"
}
```
- 鉴权：需携带微信登录获取的 JWT token
- 处理流程：
  1. 校验手机号格式
  2. 从 JWT token 中获取 `openid`
  3. 查询 `c_user` 表：
     - 先根据 `phone` 查询：`SELECT * FROM c_user WHERE phone = ? AND status = 1 AND is_deleted = 0`
     - 再根据 `openid` 查询：`SELECT * FROM c_user WHERE open_id = ? AND status = 1 AND is_deleted = 0`
  4. 处理逻辑：
     - 情况1：手机号存在且 openid 不存在 → 将 openid 关联到该手机号用户
     - 情况2：手机号不存在但 openid 存在（临时用户） → 更新手机号和用户信息
     - 情况3：手机号和 openid 都存在 → 验证是否为同一用户，是则返回成功，否则返回错误
     - 情况4：手机号和 openid 都不存在 → 创建新用户（默认角色为经纪人）
  5. 更新用户信息：`phone`、`nickname`、`headimgurl`、`wechat_bound=1`、`updated_at`
  6. 记录操作日志到 `c_operation_log`
  7. 返回完整的 JWT token
- 返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "xxxx",
    "user_info": {
      "id": 1,
      "nickname": "张三",
      "username": "张三",
      "phone": "13800000000",
      "role": 1,
      "status": 1,
      "wechat_bound": 1
    }
  }
}
```
- 错误码：
  - `code = 18`：手机号已被其他微信账号绑定
  - `code = 21`：手机号格式错误

## 线索管理（管理员专用）

### 新增线索
- POST `/api/v1/lead`
- 请求参数：
```json
{
  "customer_name": "张三",
  "customer_phone": "13800000000",
  "car_brand": "Toyota",
  "car_model": "Camry",
  "notes": "车况良好",
  "user_id": 1
}
```
- 返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1
  }
}
```

### 查询线索列表
- GET `/api/v1/lead?status=待跟进&agent_id=1&keyword=张三&page=1&size=20`
- 参数说明：
  - `status`：线索状态（可选）
  - `agent_id`：经纪人ID（可选）
  - `keyword`：搜索关键词（客户姓名，可选）
  - `page`：页码（默认1）
  - `size`：每页数量（默认20）

### 编辑线索
- PUT `/api/v1/lead/:id`
- 请求参数：
```json
{
  "customer_name": "张三",
  "customer_phone": "13800000000",
  "car_brand": "Toyota",
  "car_model": "Camry",
  "notes": "车况良好",
  "user_id": 1
}
```
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```

### 删除线索
- DELETE `/api/v1/lead/:id`
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```

### 修改线索状态
- PUT `/api/v1/lead/:id`
- 请求参数：
```json
{
  "status": 4,
  "fail_reason": "价格未谈拢",
  "car_actual_price": 85000
}
```
- 状态流转规则：
  - 待跟进(0) → 跟进中(1) → 已看车(2) → 已报价(3) → 已成交(4)/已失败(5)
  - 已成交(4) 和 已失败(5) 为终态，不可再修改
  - 状态改为已成交(4) 时，必须提供 `car_actual_price`
  - 状态改为已失败(5) 时，必须提供 `fail_reason`
  - 状态改为已成交(4) 后，系统自动生成待结算记录
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```
- 错误码：
  - `code = 10`：状态流转非法
  - `code = 11`：缺少必填参数（成交价格/失败原因）
  - `code = 12`：终态不可修改

### 查看线索详情

## 结算管理（管理员专用）

### 新增结算
- POST `/api/v1/settlement`
- 请求参数：
```json
{
  "lead_id": 1,
  "profit": 10000,
  "agent_share": 3000,
  "remark": "公司已售出"
}
```
- 业务规则：
  - 仅当关联线索状态为已成交(4)时才可创建结算
  - `agent_share` 必须小于等于 `profit`
  - 一个线索只能有一条结算记录
- 返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1
  }
}
```
- 错误码：
  - `code = 13`：线索未成交，无法创建结算
  - `code = 14`：分成金额不能大于利润金额
  - `code = 15`：该线索已存在结算记录

### 编辑结算
- PUT `/api/v1/settlement/:id`
- 请求参数：
```json
{
  "profit": 12000,
  "agent_share": 3600,
  "remark": "价格调整"
}
```
- 业务规则：
  - 仅待结算(status=0)的记录可修改金额
  - 已结算(status=1)的记录仅允许修改备注
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```
- 错误码：
  - `code = 16`：已结算记录不可修改金额

### 删除结算
- DELETE `/api/v1/settlement/:id`
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```

### 更新结算状态
- PUT `/api/v1/settlement/:id`
- 请求参数：
```json
{
  "status": 1,
  "settled_at": "2026-04-21 12:00:00"
}
```
- 业务规则：
  - 待结算(0) → 已结算(1)
  - 已结算(status=1)的记录不可再修改金额，仅允许修改备注
  - 标记为已结算时，自动触发微信服务通知（如已配置）
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```

### 查看结算详情
- GET `/api/v1/settlement/:id`
- 返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "lead_id": 1,
    "customer_name": "张三",
    "car_brand": "Toyota",
    "car_model": "Camry",
    "profit": 10000,
    "agent_share": 3000,
    "status": "待结算",
    "settled_at": null,
    "remark": "公司已售出",
    "created_at": "2026-04-21 12:00:00",
    "updated_at": "2026-04-21 12:00:00"
  }
}
```

### 查询结算列表
- GET `/api/v1/settlement?status=待结算&agent_id=1&page=1&size=20`
- 参数说明：
  - `status`：结算状态（可选）
  - `agent_id`：经纪人ID（可选）
  - `page`：页码（默认1）
  - `size`：每页数量（默认20）

### 推送结算提醒
- POST `/api/v1/settlement/:id/notify`
- 功能：触发微信服务通知，推送给对应经纪人
- 幂等性：同一结算记录 5 分钟内重复推送只发送一次通知
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```
- 错误码：
  - `code = 17`：5 分钟内已推送过提醒

## 经纪人管理（管理员专用）

### 新增经纪人
- POST `/api/v1/user`
- 功能：管理员预创建经纪人账号，支持微信端和PC端登录
- 请求参数：
```json
{
  "userid": "lisi001",
  "username": "李四",
  "phone": "13900000000",
  "password": "123456"  // 初始密码，用户可登录后修改
}
```
- 业务规则：
  - `userid` 由管理员指定，需全局唯一
  - `phone` 必须唯一，用于后续微信登录绑定
  - `password` 为初始密码，使用 bcrypt 加密存储
  - 新创建的用户 `openid` 为空，`status` 为启用，`role` 为 1（经纪人）
  - 记录操作日志到 `c_operation_log`
- 返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1
  }
}
```
- 错误码：
  - `code = 19`：userid 已存在
  - `code = 20`：手机号已存在

### 查询经纪人列表
- GET `/api/v1/user?page=1&size=20`

### 查询经纪人列表（用于选择器）
- GET `/api/v1/user/agent-list`
- 功能：返回所有启用状态的经纪人列表，用于线索表单选择
- 返回：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "username": "李四",
        "phone": "13900000000"
      }
    ]
  }
}
```

### 修改经纪人信息
- PUT `/api/v1/user/:id`
- 请求参数：
```json
{
  "status": "disabled"
}
```
- 返回：
```json
{
  "code": 0,
  "message": "success"
}
```

## 数据导出（管理员专用）

### 导出线索列表
- GET `/api/v1/lead/export?status=4&start_date=2026-04-01&end_date=2026-04-30`
- 功能：导出线索数据为 Excel 文件
- 响应：`Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- 导出字段：客户姓名、客户电话、车辆品牌、车辆型号、预估金额、实际成交金额、状态、归属经纪人、创建时间

### 导出结算列表
- GET `/api/v1/settlement/export?status=1&start_date=2026-04-01&end_date=2026-04-30`
- 功能：导出结算数据为 Excel 文件
- 响应：`Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- 导出字段：客户姓名、车辆信息、利润金额、经纪人分成、结算状态、结算时间、经纪人姓名

## 经纪人端接口（微信端+PC端）

### 我的线索
- GET `/api/v1/user/lead?status=4&keyword=张三&page=1&size=20`
- 数据权限：
  - 仅返回当前登录经纪人（`user_id = 当前用户userid`）且 `is_deleted = 0` 的线索

  - `size`：每页数量（默认20）
- 错误码：
  - `code = 22`：请先绑定手机号

### 我的结算

- 数据权限：
  - 仅返回当前登录经纪人关联线索的结算记录，且 `is_deleted = 0`
  - 未绑定手机号的用户无法访问，返回错误码 22
- 参数说明：
  - `status`：结算状态（可选）
  - `page`：页码（默认1）
  - `size`：每页数量（默认20）
- 错误码：
  - `code = 22`：请先绑定手机号

### 数据概览
- GET `/api/v1/user/overview`
- 数据权限：
  - 仅统计当前登录经纪人自己的数据
  - 未绑定手机号的用户无法访问，返回错误码 22
- 返回：
```json
{
  "total_leads": 10,

  "pending_amount": 6000,  // 待结算金额
  "settled_amount": 9000  // 已结算金额

```
- 错误码：
  - `code = 22`：请先绑定手机号

## 错误码规范
- `code = 0` → 成功
- `code = 1` → 参数错误
- `code = 2` → 鉴权失败
- `code = 3` → 数据不存在
- `code = 4` → 系统错误
- `code = 10` → 状态流转非法
- `code = 11` → 缺少必填参数（成交价格/失败原因）
- `code = 12` → 终态不可修改
- `code = 13` → 线索未成交，无法创建结算
- `code = 14` → 分成金额不能大于利润金额
- `code = 15` → 该线索已存在结算记录
- `code = 16` → 已结算记录不可修改金额
- `code = 17` → 5 分钟内已推送过提醒
- `code = 18` → 手机号已被其他微信账号绑定
- `code = 19` → userid 已存在
- `code = 20` → 手机号已存在
- `code = 21` → 手机号格式错误
- `code = 22` → 请先绑定手机号
- `code = 429` → 请求过于频繁

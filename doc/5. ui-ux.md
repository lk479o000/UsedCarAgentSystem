# UI/UX 设计说明

## 总体设计原则
- **极简操作**：所有功能尽量通过下拉选择 + 单击完成
- **数据实时同步**：后台改动即时反映到小程序
- **权限隔离**：管理员全量权限，经纪人仅能查看本人数据
- **风格统一**：
  - 后台：浅色背景 + 深色文字，使用 Vue3 默认风格
  - 小程序：玻璃拟态设计 + 原子化样式，现代化移动端体验
- **响应式设计**：
  - 后台：兼容桌面和移动端浏览器
  - 小程序：适配不同屏幕尺寸
- **性能优先**：确保小程序启动速度快，页面切换流畅

### 页面结构
- **导航栏**：线索管理、结算管理、经纪人管理、统计报表
- **首页**：今日待跟进、需回访、待结算提醒（卡片式展示）
- **线索列表**：
  - 表格展示：客户姓名、电话、车辆信息、状态、归属经纪人
  - 筛选条件：状态、经纪人、时间范围
  - 操作按钮：修改状态、查看详情
- **结算管理**：
  - 列表：线索信息、利润、分成金额、状态
  - 操作：标记已结算、填写结算时间
  - 统计：全局统计 + 按经纪人统计
- **经纪人管理**：
  - 列表：姓名、手机号、状态、业务数据
  - 操作：新增、编辑、禁用
- **密码修改页面**：
  - 表单结构：原密码、新密码、确认密码
  - 密码强度实时检测：
    - 密码强度条：根据密码强度显示不同颜色（弱-红色、一般-橙色、良好-黄色、强-绿色）
    - 详细密码要求：显示具体的密码要求项及其状态
    - 实时反馈：输入密码时即时更新强度评估
  - 验证规则：
    - 密码长度至少8位
    - 必须包含字母和数字
    - 不能与原密码相同
    - 不能使用常见弱密码
- **表格组件设计**：
  - **列固定（Sticky Column）**：提供两种配置方式，意义不同
    - 固定列自动计算偏移量，支持多列连续固定
    - 固定列边界显示阴影指示，提示用户可滚动
    - 固定列背景色与表格整体一致（渐变表头 + 斑马纹内容）

    ### 方式一：组件级属性 `fixed-first-column`
    **意义**：快速固定第一列的便捷配置
    - **配置位置**：UTable 组件属性
    - **作用范围**：仅作用于索引为0的第一列
    - **适用场景**：大多数业务场景下需要固定首列（如客户姓名、ID等）
    - **优势**：无需修改列定义，一行代码搞定
    - **实现原理**：当 `fixed-first-column=true` 时，组件自动将第1列视为左固定列（见 UTable.vue 第84-85行）
    - **配置示例**：
      ```vue
      <!-- Leads.vue 第40行：使用 fixed-first-column 固定客户姓名列 -->
      <UTable :data="tableData" :columns="columns" fixed-first-column @row-click="handleRowClick" />
      ```
    - **实际应用**：
      - 线索管理列表：固定"客户姓名"列
      - 结算管理列表：固定"客户姓名"或"线索ID"列
      - 经纪人管理列表：固定"姓名"列

    ### 方式二：列定义属性 `fixed: 'left' | 'right'`
    **意义**：灵活控制任意列的固定位置
    - **配置位置**：columns 数组中的列定义对象
    - **作用范围**：可指定任意列为左侧或右侧固定列
    - **适用场景**：
      - 需要固定非首列的情况（如同时固定序号+姓名两列）
      - 需要固定右侧操作列
      - 需要多列复杂固定布局
    - **优势**：精确控制每列的固定行为，支持多列组合
    - **配置示例**：
      ```javascript
      // 固定右侧操作列（最常用）
      { key: 'action', title: '操作', width: '180px', fixed: 'right' }

      // 带对齐方式的操作列
      { key: 'action', title: '操作', width: '100px', align: 'center', fixed: 'right' }

      // 固定左侧非首列（如第2列也需要固定）
      { key: 'id', title: 'ID', width: '80px', fixed: 'left' },
      { key: 'name', title: '名称', width: '120px' },  // 被 fixed-first-column 固定
      ```

    ### 实际项目应用案例
    ```javascript
    // Leads.vue 第578-596行：管理员视图的双重固定
    const columns = computed(() => {
      const baseColumns = [
        { key: 'customerName', title: '客户姓名', width: '120px' },
        // ... 其他列 ...
      ]
      
      if (userStore.isAdmin) {
        baseColumns.push({ key: 'agent', title: '归属经纪人', width: '120px' })
        baseColumns.push({ key: 'action', title: '操作', width: '180px', fixed: 'right' })  // 列级固定
      }
      
      return baseColumns
    })
    
    // 模板中使用组件级固定（Leads.vue 第40行）
    <UTable :data="tableData" :columns="columns" fixed-first-column />
    // 结果：customerName 列被 fixed-first-column 固定 + action 列被 fixed:'right' 固定
    ```

    ### 两种方式对比

    | 对比项 | `fixed-first-column` | `fixed: 'left/right'` |
    |--------|---------------------|----------------------|
    | **配置层级** | 组件属性 | 列定义对象 |
    | **灵活性** | 低（仅限首列） | 高（任意列） |
    | **修改成本** | 无需改列定义 | 需修改 columns |
    | **典型场景** | 首列固定需求 | 复杂固定布局 |
    | **可叠加** | ✅ 可与 fixed:'right' 叠加 | ✅ 可与其他 fixed 叠加 |
    | **推荐度** | ⭐⭐⭐⭐⭐（首选） | ⭐⭐⭐⭐（按需） |

    ### 最佳实践建议
    1. **优先使用 `fixed-first-column`**：90%的场景只需固定首列，这是最简洁的方式
    2. **配合 `fixed: 'right'` 使用**：首列 + 操作列的双固定是标准配置
    3. **仅在特殊场景用 `fixed: 'left'`**：当需要固定多个左侧列时才使用
    4. **必须设置 width**：所有固定列都需要明确的宽度值
    5. **宽度建议**：
       - 操作列：100-200px（根据按钮数量调整）
       - ID/序号列：60-80px
       - 姓名列：100-150px
  - **列对齐**：支持 `align: 'left'` / `'center'` / `'right'` 配置
    - 文本/名称类：左对齐（默认）
    - 金额/数字类：右对齐
    - 状态/标签/操作类：居中对齐
    - 表头与内容对齐方式保持一致
  - **操作列设计**：
    - 使用 `UActionGroup` 组件管理操作按钮
    - 支持 `max` 属性控制最多显示按钮数（默认2个），超出自动收入"更多"下拉菜单
    - 按钮点击不触发行点击事件（阻止事件冒泡）
    - 操作列宽度自适应，不设置固定宽度，由内容撑开
    - 配置示例：`<UActionGroup :actions="getActions(row)" :max="2" />`
  - **响应式适配**：
    - 表格容器支持水平滚动
    - 列宽可配置，建议关键列设置固定宽度，操作列不设置宽度由内容自适应

## 微信小程序（ucar-miniapp）
### 设计系统
- **颜色体系**：
  - 主色：#3B82F6（蓝色）
  - 辅色：#10B981（绿色）、#F59E0B（黄色）、#EF4444（红色）
  - 中性色：#F3F4F6（背景）、#E5E7EB（边框）、#6B7280（文本）、#1F2937（标题）
- **样式系统**：
  - 原子化样式：使用 `unocss-preset-weapp`
  - 玻璃拟态：`card-glass` 类实现半透明效果
  - 渐变按钮：`btn-primary` 类实现蓝白渐变
  - 语义化标签：`tag-success`、`tag-warning`、`tag-danger` 等
- **图标系统**：
  - 使用 `@unocss/preset-icons` + Lucide 图标集
  - 统一图标风格，确保视觉一致性
- **字体系统**：
  - 标题：16-20px，粗体
  - 正文：14px，常规
  - 辅助文字：12px，浅色
- **间距系统**：
  - 卡片间距：16px
  - 元素间距：8-12px
  - 页面边距：16px

### 页面结构

#### 1. 登录页
- 微信一键登录按钮
- 手机号绑定表单
- 用户协议和隐私政策勾选
- 加载状态反馈

#### 2. 首页
- **数据概览**：
  - 总线索数、已成交、已失败（数字卡片）
  - 待结算金额、已结算金额（金额卡片）
- **快捷入口**：
  - 我的线索、我的结算、个人中心（图标+文字）
  - 刷新数据按钮
- **最近线索**：
  - 列表展示（客户姓名、车辆信息、状态）
  - 点击进入详情
- **下拉刷新**：
  - 顶部刷新动画
  - 数据实时更新

#### 3. 线索管理
- **线索列表**：
  - 7状态筛选：全部/待跟进/跟进中/已看车/已报价/已成交/已失败（横向滚动）
  - 搜索功能：按客户姓名搜索（顶部搜索框）
  - 列表项：
    - 客户姓名、车辆信息、状态标签
    - 经纪人视角：客户电话中间4位脱敏
    - 管理员视角：额外显示归属经纪人
  - 下拉刷新 + 上拉加载
  - 管理员：右下角悬浮新增按钮
- **线索详情**：
  - **基本信息**：客户姓名/电话/类型/车辆信息/状态/归属经纪人/备注/创建时间
  - **跟进记录**（仅管理员可见）：
    - 列表展示（内容、时间）
    - 新增/编辑/删除操作
  - **底部操作栏**（仅管理员可见）：
    - 编辑：进入编辑表单
    - 更新状态：底部ActionSheet选择新状态（含成交价格/失败原因输入）
    - 结算/重新结算：进入结算表单
    - 删除：确认弹窗后删除
- **线索表单**：
  - 新增/编辑线索
  - 字段：客户姓名、客户电话、客户类型、车辆品牌、车辆型号、归属经纪人、备注
  - 表单验证：必填项检查、手机号格式验证
  - 归属经纪人选择器：从经纪人列表中选择

#### 4. 结算管理
- **结算列表**：
  - 状态筛选：全部/待结算/已结算（横向滚动）
  - 列表项：
    - 客户姓名、车辆信息、利润金额、经纪人分成、状态
  - 下拉刷新 + 上拉加载
  - 管理员：右下角悬浮新增按钮
  - 点击卡片进入详情
- **结算详情**：
  - **基本信息**：客户姓名/车辆信息/利润金额/经纪人分成/状态/结算时间/备注/创建时间
  - **底部操作栏**（仅管理员可见）：
    - 编辑：进入编辑表单
    - 更新状态：切换待结算/已结算（含结算时间选择）
- **结算表单**：
  - 新增/编辑/重新结算
  - 字段：关联线索、利润金额、经纪人分成、备注
  - 表单验证：关联线索必填、利润≥0、分成≥0且≤利润
  - 线索选择：跳转线索列表选择已成交线索

#### 5. 个人中心
- **用户信息**：
  - 微信头像、昵称、姓名、角色、状态、手机号
  - 管理员/经纪人标识
- **功能菜单**：
  - 修改密码（含强度检测）
  - 绑定手机号
  - 用户协议
  - 隐私政策
  - 退出登录
- **设置项**：
  - 清除缓存
  - 关于我们
  - 版本信息

#### 6. 协议页面
- 用户协议
- 隐私政策
- 可滚动阅读
- 底部返回按钮

### 交互设计

  - 下拉刷新：更新数据
  - 上拉加载：加载更多

  - 确认操作：确认弹窗

## 交互流程示例
1. **线索管理流程**：
   - 管理员登录 → 新增线索 → 状态流转（待跟进 → 跟进中 → 已看车 → 已报价 → 已成交）→ 生成结算
2. **结算管理流程**：
   - 管理员标记结算为已结算 → 推送微信提醒 → 经纪人收到提醒 → 点击进入详情页 → 查看收益
3. **经纪人操作流程**：
   - 微信登录 → 绑定手机号 → 查看首页概览 → 查看线索列表 → 查看结算列表 → 个人中心管理
4. **权限控制流程**：
   - 管理员：完整操作权限，可新增/编辑/删除
   - 经纪人：只读权限，只能查看自己的数据

## 可用性要求
- **操作效率**：所有操作 ≤ 3 次点击完成
- **错误提示**：错误提示需明确（如手机号已存在、状态不可更改）
- **数据展示**：列表支持分页与搜索，确保数据清晰易读
- **网络适应**：小程序端需保证在低网速下仍可正常加载数据
- **响应速度**：页面切换时间 ≤ 1s，操作反馈时间 ≤ 0.5s
- **可访问性**：字体大小适中，颜色对比度符合标准，操作区域足够大

## 性能优化
- **小程序启动**：
  - 代码包大小控制在 2MB 以内
  - 首屏数据预加载
  - 图片资源懒加载
- **页面渲染**：
  - 减少不必要的 `setData` 调用
  - 使用虚拟列表处理长列表
  - 避免复杂的 WXML 结构
- **网络请求**：
  - 合并请求，减少网络往返
  - 合理使用缓存
  - 接口响应时间优化

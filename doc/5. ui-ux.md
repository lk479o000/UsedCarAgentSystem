# UI/UX 设计说明

## 总体设计原则
- 极简操作：所有功能尽量通过下拉选择 + 单击完成
- 数据实时同步：后台改动即时反映到小程序
- 权限隔离：管理员全量权限，经纪人仅能查看本人数据
- 风格统一：浅色背景 + 深色文字，后台使用 Vue3 默认风格，小程序使用微信原生组件
- 响应式设计：后台前端需兼容桌面和移动端浏览器

---

## 管理后台（ucar-admin-web）
### 页面结构
- 导航栏：线索管理、结算管理、经纪人管理、统计报表
- 首页：今日待跟进、需回访、待结算提醒（卡片式展示）
- 线索列表：
  - 表格展示：客户姓名、电话、车辆信息、状态、归属经纪人
  - 筛选条件：状态、经纪人、时间范围
  - 操作按钮：修改状态、查看详情
- 结算管理：
  - 列表：线索信息、利润、分成金额、状态
  - 操作：标记已结算、填写结算时间
  - 统计：全局统计 + 按经纪人统计
- 经纪人管理：
  - 列表：姓名、手机号、状态、业务数据
  - 操作：新增、编辑、禁用
- 密码修改页面：
  - 表单结构：原密码、新密码、确认密码
  - 密码强度实时检测：
    - 密码强度条：根据密码强度显示不同颜色（弱-红色、一般-橙色、良好-黄色、强-绿色）
    - 详细密码要求：显示具体的密码要求项及其状态
    - 实时反馈：输入密码时即时更新强度评估
  - 验证规则：
    - 密码长度至少8位
    - 必须包含字母和数字
    - 不能与原密码相同
    - 不能使用常见弱密码

## 微信小程序（ucar-miniapp）
### 页面结构
- 首页：
  - 数据概览：总线索数、成交数、失败数、待结算金额、已结算金额
  - 快捷入口：我的线索、我的结算
- 我的线索：
  - 列表：客户姓名、车辆信息、状态
  - 详情：跟进记录、失败原因（只读）
- 我的结算：
  - 待结算：车辆信息、分成金额、待结算原因
  - 已结算：车辆信息、分成金额、结算时间
  - 统计：累计已结算收益、累计待结算收益
- 结算提醒：
  - 微信服务通知 → 点击进入结算详情页
- 个人中心：
  - 展示姓名、手机号
  - 操作：退出登录

---

## 交互流程示例
1. 管理员录入线索 → 经纪人首页数据更新
2. 管理员标记线索为已成交 → 生成结算记录
3. 管理员标记结算为已结算 → 推送微信提醒
4. 经纪人收到提醒 → 点击进入详情页 → 查看收益

---

## 可用性要求
- 所有操作 ≤ 3 次点击完成
- 错误提示需明确（如手机号已存在、状态不可更改）
- 列表支持分页与搜索
- 小程序端需保证在低网速下仍可正常加载数据

# 二手车收车经纪人系统 - 需求说明书

## 项目目标
- 管理员：规范线索管理、结算流程，避免遗漏
- 经纪人：透明化收益，实时结算提醒，减少沟通成本

## 核心角色
- **用户**（通过role字段区分：0=管理员, 1=经纪人）
  - **管理员**：微信小程序操作 + PC后台操作，负责线索管理、结算管理、用户管理
  - **经纪人**：微信小程序操作，负责查看线索、查看结算、接收结算提醒

## 功能需求

### 管理后台功能
- **登录**：账号密码+验证码登录
- **线索管理**：
  - 新增线索
  - 查看线索列表（支持筛选和搜索）
  - 修改线索状态
  - 查看线索详情
  - 管理跟进记录
- **结算管理**：
  - 新增结算
  - 查看结算列表（支持筛选）
  - 更新结算状态
  - 推送结算提醒
- **用户管理**：
  - 新增经纪人
  - 查看经纪人列表
  - 修改经纪人信息
  - 管理经纪人状态

### 微信小程序功能
- **登录**：
  - 微信一键登录
  - 手机号绑定
- **首页概览**：
  - 数据概览（总线索、已成交、已失败）
  - 收益统计（待结算金额、已结算金额）
  - 最近线索列表
  - 快捷入口
- **线索管理**：
  - 线索列表（7状态筛选：全部/待跟进/跟进中/已看车/已报价/已成交/已失败）
  - 线索详情（基本信息、跟进记录）
  - 线索搜索（客户姓名）
  - 管理员：新增/编辑/删除线索、管理跟进记录
- **结算管理**：
  - 结算列表（待结算/已结算筛选）
  - 结算详情（基本信息、状态更新）
  - 管理员：新增/编辑/删除结算
- **个人中心**：
  - 用户信息展示（头像、昵称、姓名、角色、状态、手机号）
  - 修改密码（含强度检测）
  - 绑定手机号
  - 协议与隐私政策
  - 退出登录

## 交互逻辑
- 管理员登录（账号密码+验证码）→ 录入线索 → 状态流转 → 成交 → 结算 → 推送提醒 → 经纪人查看收益
- 经纪人微信登录 → 查看线索和结算信息

## 技术需求
- **后端**：Node.js + Express
- **PC端前端**：Vue3
- **小程序**：微信原生小程序
- **数据库**：MySQL 8+
- **样式**：
  - PC端：UnoCSS
  - 小程序：unocss-preset-weapp（原子化样式）
- **图标**：Lucide 图标集

## 权限需求
- **管理员**：完整操作权限
- **经纪人**：只读权限，只能查看自己的数据

## 安全需求
- JWT 鉴权
- 手机号脱敏
- 密码强度检测
- 接口频率限制

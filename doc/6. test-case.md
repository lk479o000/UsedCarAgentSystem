# 测试用例设计

## 登录
- 输入正确管理员账号 → 成功登录，返回 token
- 输入错误密码 → 登录失败，返回错误码
- 输入正确经纪人手机号 + 验证码 → 成功登录
- 输入未录入手机号 → 登录失败

---

## 线索管理（管理员）
- 新增线索 → 列表显示新增数据
- 修改状态为"已成交" → 自动生成结算记录
- 修改状态为"已失败" → 必须填写失败原因
- 查询线索列表 → 支持分页与筛选
- 异常：重复客户手机号 → 返回错误提示

---

## 结算管理（管理员）
- 成交线索 → 进入待结算列表
- 标记已结算 → 状态更新成功，推送提醒
- 查看统计 → 数据正确（总利润、分成金额）
- 异常：结算金额为负数 → 返回错误提示
- 异常：未成交线索尝试结算 → 返回错误提示

---

## 经纪人管理（管理员）
- 新增经纪人 → 列表显示新增数据
- 修改经纪人状态为 disabled → 经纪人无法登录
- 异常：手机号重复 → 返回错误提示

---

## 经纪人端（小程序）
- 查看我的线索 → 列表显示本人线索
- 查看我的结算 → 待结算/已结算列表正确
- 收到结算提醒 → 点击进入详情页
- 数据概览 → 显示正确的统计数据

---

## 安全测试
- 未登录状态访问接口 → 返回鉴权失败
- 经纪人尝试访问管理员接口 → 返回越权错误
- 管理员尝试访问经纪人专用接口 → 返回错误提示

---

## 性能测试
- 查询线索列表（1000条数据） → 响应时间 < 200ms
- 并发请求（50个同时登录） → 系统正常响应

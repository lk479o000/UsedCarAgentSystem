# 测试用例设计

## 登录
- 输入正确管理员账号+密码+验证码 → 成功登录，返回 token（PC端）
- 输入错误密码 → 登录失败，返回错误码
- 验证码错误/过期 → 登录失败，返回错误码
- 连续 5 次密码错误 → 账号锁定 30 分钟
- 管理员微信登录 → 成功返回 token（微信端）
- 已绑定手机号的经纪人微信登录 → 直接返回 token（微信端）
- 未绑定手机号的用户微信登录 → 创建临时用户，返回 need_bind_phone=true
- 经纪人使用账号密码登录 → 成功返回 token（PC端）
- 绑定手机号时手机号已存在且未绑定微信 → 将微信关联到该手机号用户，返回 token
- 绑定手机号时手机号不存在但微信已存在（临时用户） → 更新手机号，返回 token
- 绑定手机号时手机号和微信都存在且为同一用户 → 返回成功
- 绑定手机号时手机号和微信都存在但为不同用户 → 返回错误码 18
- 绑定手机号时手机号和微信都不存在 → 创建新用户，返回 token
- 绑定手机号时手机号格式错误 → 返回错误码 21
- 禁用状态的用户微信登录 → 返回账号已禁用错误
- 禁用状态的用户账号密码登录 → 返回账号已禁用错误

## 线索管理（管理员）
- 新增线索 → 列表显示新增数据
- 新增线索时客户手机号已存在 → 返回错误提示（是否允许重复需根据业务确认）
- 修改状态：待跟进(0) → 跟进中(1) → 已看车(2) → 已报价(3) → 已成交(4) → 成功
- 修改状态为已成交(4) 时未提供 car_actual_price → 返回错误码 11
- 修改状态为已失败(5) 时未提供 fail_reason → 返回错误码 11
- 修改状态：已成交(4) → 已失败(5) → 返回错误码 12（终态不可修改）
- 修改状态：已报价(3) → 待跟进(0) → 返回错误码 10（状态流转非法）
- 状态改为已成交(4) 后 → 自动创建待结算记录
- 查询线索列表 → 支持分页与筛选，page/size 超出限制时按最大值处理
- 删除线索 → 软删除，is_deleted=1，关联结算保留
- 尝试删除已成交线索 → 返回错误（已成交不可删除）

---

## 结算管理（管理员）
- 成交线索 → 进入待结算列表
- 新增结算时 lead_id 对应线索未成交 → 返回错误码 13
- 新增结算时 agent_share > profit → 返回错误码 14
- 新增结算时该线索已有结算记录 → 返回错误码 15
- 标记已结算 → 状态更新成功，自动推送微信提醒
- 已结算记录尝试修改 profit/agent_share → 返回错误码 16
- 已结算记录修改 remark → 成功
- 5 分钟内重复推送同一结算提醒 → 返回错误码 17
- 查看统计 → 数据正确（总利润、分成金额）
- 结算金额为负数/零 → 返回错误提示
- 导出结算列表 → 生成 Excel 文件，字段完整

---

## 经纪人管理（管理员）
- 新增经纪人 → 列表显示新增数据，openid 为空，密码已加密存储
- 新增经纪人时手机号重复 → 返回错误码 20
- 新增经纪人时 userid 重复 → 返回错误码 19
- 新增经纪人时未提供密码 → 返回参数错误
- 经纪人修改自己的密码 → 成功
- 经纪人修改密码时旧密码错误 → 返回错误提示
- 修改经纪人状态为禁用(0) → 经纪人无法登录
- 禁用经纪人 → 其已有线索和结算保留，但不可归属新线索
- 删除经纪人 → 软删除，is_deleted=1

---

## 经纪人端（微信端+PC端）
- 微信端：已绑定手机号的用户查看我的线索 → 列表仅显示本人线索且 is_deleted=0
- 微信端：已绑定手机号的用户查看我的结算 → 待结算/已结算列表正确，仅显示本人数据
- 微信端：已绑定手机号的用户查看数据概览 → 显示正确的统计数据
- 微信端：未绑定手机号的用户查看我的线索 → 返回错误码 22
- 微信端：未绑定手机号的用户查看我的结算 → 返回错误码 22
- 微信端：未绑定手机号的用户查看数据概览 → 返回错误码 22
- 微信端：收到结算提醒 → 点击进入详情页
- PC端：经纪人使用账号密码登录 → 成功进入管理后台
- PC端：已绑定手机号的用户查看我的线索 → 列表仅显示本人线索且 is_deleted=0
- PC端：已绑定手机号的用户查看我的结算 → 待结算/已结算列表正确，仅显示本人数据
- PC端：已绑定手机号的用户查看数据概览 → 显示正确的统计数据
- 禁用状态的经纪人登录 → 返回账号已禁用错误
- 尝试查看其他经纪人的线索 → 返回空列表或无权访问

---

## 线索导入
- 上传 Excel 文件 → 成功解析并存储到 c_lead_import
- 导入时保留原始文本格式 → 数据与 Excel 一致
- 同一批次重复导入 → 根据批次号区分，不覆盖已有数据
- 查看导入记录列表 → 按批次分组显示
- 将导入记录转为线索 → 创建 c_lead 记录，更新 import_status=1
- 忽略导入记录 → 更新 import_status=2
- 转为线索时未指定归属经纪人 → 返回错误提示
- 重复客户提示 → 根据 customer_name + customer_phone 检测重复

---

## 安全测试
- 未登录状态访问接口 → 返回鉴权失败 (code=2)
- 使用过期/伪造 token 访问接口 → 返回鉴权失败
- 经纪人尝试访问管理员接口 → 返回越权错误
- 管理员尝试访问经纪人专用接口 → 返回错误提示
- 接口频率限制测试：同一 IP 快速请求登录接口 → 返回 429
- SQL 注入测试：在查询参数中传入特殊字符 → 正常过滤，不报错
- XSS 测试：在备注字段输入脚本标签 → 存储时转义，输出时安全

---

## 性能测试
- 查询线索列表（1000条数据） → 响应时间 < 200ms
- 查询线索列表（10000条数据，分页 size=100） → 响应时间 < 500ms
- 并发请求（50个同时登录） → 系统正常响应，无死锁
- 并发请求（100个同时查询列表） → 系统正常响应
- 大数据量统计查询 → 响应时间 < 1000ms
- Excel 导入（1000 行数据） → 响应时间 < 5000ms

---

## 边界值测试
- 手机号格式校验：11位数字、非11位、包含字母、空值
- 金额字段：最大值、0、负数、超大值
- 字符串字段：空值、最大长度、超长（超过数据库限制）
- 分页参数：page=0、page=-1、size=0、size=1000、size=-10
- 状态字段：非法值（如 99、-1、字符串）
- Excel 导入：空文件、格式错误、超大文件（>10MB）

---

## 异常场景测试
- 数据库连接断开 → 返回系统错误 (code=4)，不暴露堆栈信息
- 微信服务不可用 → 微信登录返回友好错误提示
- 文件导出时数据量过大 → 异步生成，提供下载链接
- Excel 导入时列名不匹配 → 返回清晰的错误提示

## 一、现状分析

### 1.1 现有页面

| 页面 | 路径 | 功能 |
|------|------|------|
| 线索列表 | `pages/leads/leads` | 状态折叠筛选 + 搜索 + 卡片列表 + FAB新增 |
| 线索详情 | `pages/leads/detail` | 基本信息卡片 + 跟进记录列表 + 底部操作栏 |
| 线索表单 | `pages/leads/form` | 新增/编辑线索，表单字段平铺 |
| 跟进表单 | `pages/followups/form` | 新增/编辑跟进记录 |

### 1.2 核心痛点

1. **跟进记录缺乏时间轴感**：当前为简单卡片堆叠（`bg-gray-50 rounded-xl p-4 mb-3`），无法直观感受跟进节奏和时间跨度
2. **详情页信息层级扁平**：状态、价格等关键信息不够突出，所有信息平铺在一个卡片内
3. **经纪人无法看到跟进记录**：跟进记录区域加了 `wx:if="{{isAdmin}}"`，经纪人视角下完全不可见
4. **状态流转缺乏可视化**：6个状态仅用一个标签展示，无法直观感知当前进度和后续路径
5. **列表页缺少跟进摘要**：需进入详情才能看到跟进动态，无法在列表快速判断优先级
6. **操作栏占用空间大**：底部固定操作栏遮挡内容，且操作项过多（编辑/更新状态/结算/删除）

### 1.3 数据模型回顾

**线索状态流转**：
```
待跟进(0) → 跟进中(1) → 已看车(2) → 已报价(3) → 已成交(4)
                                           ↘ 已失败(5)
```

**跟进记录字段**：
- `followup_content` — 跟进内容（必填）
- `followup_result` — 跟进结果（可选）
- `followup_time` — 这次跟进时间（必填）
- `next_followup_time` — 下次跟进时间（可选）
- `operator_user_id` — 操作人（关联 User）

### 2.2 改造要点

#### 2.2.1 状态Tab替代折叠面板

**现状**：折叠面板需点击展开 → 选择状态 → 收起，操作步骤多。

**改为**：横向滚动Tab栏，点击即筛选，右侧显示各状态数量。

```
全部(42)  待跟进(5)  跟进中(12)  已看车(8)  已报价(6)  已成交(9)  已失败(2)
 ────────
 选中态：下划线 + 主色文字 + 加粗
```

- 点击Tab立即筛选，无需展开/收起
- 数量徽标实时显示各状态线索数
- 横向可滚动，支持7个状态Tab
- 当前选中Tab高亮，其余灰色

#### 2.2.2 卡片增加跟进摘要区

**现状**：卡片仅显示客户名、车辆、区域、时间。

**改为**：底部增加分割线 + 最近跟进摘要 + 下次跟进时间。

```xml
<!-- 新增跟进摘要区域 -->
<view class="followup-summary" wx:if="{{item.lastFollowupContent}}">
  <view class="divider"></view>
  <view class="summary-row">
    <view class="i-lucide-message-circle text-xs text-gray-400"></view>
    <text class="text-xs text-gray-500 truncate">{{item.lastFollowupContent}}</text>
  </view>
  <view class="summary-row" wx:if="{{item.nextFollowupTime}}">
    <view class="i-lucide-clock text-xs text-primary"></view>
    <text class="text-xs text-primary">下次跟进：{{item.nextFollowupTime}}</text>
  </view>
</view>
```

- 跟进摘要最多显示1行，超出截断
- 下次跟进时间超期时显示红色提醒
- 无跟进记录时不显示该区域

#### 2.2.3 超期跟进提醒

下次跟进时间已过但状态仍为「待跟进/跟进中」的线索，卡片左侧加红色竖条：

```
▌ 张三  [买家] [跟进中]     ← 红色竖条表示超期
▌ 宝马 3系
▌ 📅 下次跟进：昨天 14:00 ⚠️
```

#### 2.2.4 搜索框简化

**现状**：经纪人和客户两个独立搜索框，占空间。

**改为**：单个搜索框，输入内容同时匹配客户姓名和手机号。管理员模式下增加经纪人搜索（可折叠）。

#### 2.2.5 其他交互

| 交互 | 说明 |
|------|------|
| 下拉刷新 | 原生下拉刷新 + 骨架屏占位 |
| 上拉加载 | 分页加载，底部渐变loading |
| 卡片点击 | 进入线索详情页 |
| FAB新增 | 管理员可见，保持不变 |

### 2.3 API适配

列表接口需返回跟进摘要字段（需后端配合）：

```
GET /api/v1/lead 或 /api/v1/user/lead

响应新增字段：
- lastFollowupContent: string   — 最近一次跟进内容（截断至50字）
- nextFollowupTime: string      — 最近一次跟进记录的下次跟进时间
```

后端实现方式：在 `getLeadList` 查询时，关联查询最新一条跟进记录，提取 `followup_content` 和 `next_followup_time`。

### 3.2 改造要点

#### 3.2.1 状态进度条

在线索头部卡片内，增加可视化状态进度条，直观展示线索所处阶段。

**正常流转态**：
```
 ●────●────○────○────○
 待   跟进  看   报价  成交
 跟进  中   车
```
- 已完成节点：实心圆（主色 #0EA5E9）+ 实线连接
- 当前节点：实心圆 + 脉冲动画（呼吸灯效果）
- 未来节点：空心圆（灰色）+ 虚线连接

**已失败态**：
```
 ●────●────✕
 待   跟进  已失败
 跟进  中
```
- 失败节点：红色 ✕ 图标
- 失败原因折叠显示在进度条下方，点击展开

**已成交态**：
```
 ●────●────●────●────●
 待   跟进  看   报价  成交
 跟进  中   车
```
- 全部节点实心 + 主色连线
- 成交金额显示在进度条下方

#### 3.2.2 快捷操作区

**管理员可见**：
```

```

**更多菜单**（点击 ··· 展开）：

## 四、跟进记录展示方案

### 4.1 ⭐ 推荐方案：垂直时间轴 + 智能折叠

#### 4.1.1 展示形态

```
    ●  今天 14:30              ← 时间节点
    │  ┌────────────────────┐  ← 内容卡片
    │  │ 👤 管理员           │
    │  │ 客户表示对3系感兴趣，│
    │  │ 预算15万左右        │
    │  │ ✅ 结果：意向明确    │
    │  │ 📅 下次：明天 14:00 │
    │  │         [编辑][删除] │  ← 管理员可见
    │  └────────────────────┘
    │
    ●  昨天 10:00
    │  ┌────────────────────┐
    │  │ 👤 管理员           │
    │  │ 首次电话联系客户，   │
    │  │ 客户态度积极        │
    │  └────────────────────┘
    │
    ●  3天前
       线索创建
```

#### 4.1.2 智能折叠规则

| 跟进记录数 | 展示策略 |
|-----------|---------|
| 1-5条 | 全部展开 |
| 6-10条 | 最新3条展开 + 「查看更早记录」按钮 |
| >10条 | 最新3条展开 + 中间折叠（显示「还有N条记录」）+ 最早1条展示 |

点击「查看更早记录」后全部展开，按钮变为「收起」。

#### 4.1.3 时间节点格式

| 时间跨度 | 显示格式 | 示例 |
|---------|---------|------|
| 今天 | 今天 HH:mm | 今天 14:30 |
| 昨天 | 昨天 HH:mm | 昨天 10:00 |
| 2-7天前 | N天前 | 3天前 |
| 今年内 | M/D HH:mm | 3/15 09:00 |
| 跨年 | YYYY/M/D | 2025/12/1 |

#### 4.1.4 内容卡片结构

```
┌────────────────────────────┐
│ 👤 管理员         [编辑][删除]│  ← 操作人 + 操作按钮（管理员）
│                            │
│ 跟进内容文字，最多显示3行，  │  ← 跟进内容
│ 超出部分折叠...             │
│                            │
│ ✅ 结果：意向明确           │  ← 跟进结果（可选）
│ 📅 下次跟进：明天 14:00    │  ← 下次跟进时间（可选）
└────────────────────────────┘
```

- 跟进内容超3行折叠，点击展开
- 跟进结果为空时不显示该行
- 下次跟进时间超期时显示红色
- 编辑/删除按钮仅管理员可见

#### 4.1.5 空状态

```
    ┆                            ← 虚线竖线
    ○                            ← 空心圆
    │  暂无跟进记录
    │  点击下方按钮添加第一条跟进
    │
    [+ 添加跟进记录]
```

#### 4.1.6 底部添加按钮

```
┌────────────────────────────┐
│  + 添加跟进记录              │  ← 虚线边框按钮，管理员可见
└────────────────────────────┘
```

```xml
  <view class="add-followup-btn" wx:if="{{isAdmin && followups.length > 0}}" bindtap="onAddFollowup">
    <view class="i-lucide-plus text-sm text-primary"></view>
    <text class="text-sm text-primary">添加跟进记录</text>
  </view>
</view>
```

## 六、跟进记录表单页改造方案

### 6.1 现状

当前跟进表单为独立页面（`pages/followups/form`），包含：跟进内容、跟进结果、这次跟进时间、下次跟进时间。

### 6.2 改造要点

| 改造项 | 说明 |
|--------|------|
| 保持独立页面 | 不改为弹窗，因为跟进内容可能较长，需要足够输入空间 |
| 跟进内容 | 改为 textarea，最多500字，右下角显示字数统计 |
| 跟进结果 | 增加快捷选项标签：「意向明确」「需继续跟进」「暂无意向」「已约看车」「已报价」 |
| 这次跟进时间 | 默认填充当前时间，支持修改 |
| 下次跟进时间 | 增加快捷选项：「明天」「3天后」「下周」「自定义」 |

**快捷结果标签**：
```
[意向明确] [需继续跟进] [暂无意向] [已约看车] [已报价]
```
点击标签自动填入跟进结果字段，也可手动输入。

**下次跟进快捷选项**：
```
[明天] [3天后] [下周] [自定义日期]
```
点击「明天」自动计算填入日期，点击「自定义日期」弹出日期选择器。

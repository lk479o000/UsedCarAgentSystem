# AI Skill 模块分工说明

## 总体原则
- 每个 skill 对应一个开发职责模块
- 输入：文档或需求说明
- 输出：代码、配置或文档片段
- 所有 skill 可独立运行，也可协同工作

---

## Skill 列表

### ui-ux-pro-max
- 职责：负责界面设计与交互逻辑（基于Vue3）
- 输入：`ui-ux.md`
- 输出：Vue3前端页面代码、交互说明、样式文件
- 协作：与 `api-builder` 对接接口，与 `test-genius` 验证交互流程

### db-master
- 职责：负责数据库建模与 SQL 生成
- 输入：`database-schema.md`
- 输出：建表语句、索引优化脚本
- 协作：与 `api-builder` 对接数据层

### api-builder
- 职责：负责接口设计与实现
- 输入：`api-spec.md`
- 输出：后端接口代码、鉴权逻辑
- 协作：与 `db-master` 共享数据结构，与 `ui-ux-pro-max` 提供接口

### test-genius
- 职责：负责测试用例与自动化脚本
- 输入：`test-case.md`
- 输出：单元测试、接口测试脚本
- 协作：与所有 skill 验证功能正确性

### deploy-helper
- 职责：负责部署脚本与环境配置
- 输入：`deploy.md`
- 输出：Dockerfile、Nginx 配置、环境变量模板
- 协作：与 `api-builder` 部署后端，与 `ui-ux-pro-max` 部署前端

---

## 扩展技能
### log-watcher
- 职责：日志监控与告警
- 输入：运行日志
- 输出：告警通知、日志分析报告

### perf-optimizer
- 职责：性能优化与压力测试
- 输入：`test-case.md` + 系统运行数据
- 输出：优化建议、索引调整、缓存策略

### doc-writer
- 职责：生成开发文档与使用手册
- 输入：所有 `.md` 文件
- 输出：统一的 README、API 文档、用户手册

---

## 优先级说明
- 初期必备：`db-master`、`api-builder`、`ui-ux-pro-max`
- 中期必备：`test-genius`、`deploy-helper`
- 后期优化：`log-watcher`、`perf-optimizer`、`doc-writer`

# 汽车经纪人线索管理系统

## 项目简介

汽车经纪人线索管理系统是一个为汽车经纪人提供线索管理、结算管理等功能的系统，包括PC端管理后台和微信小程序端。

## 技术栈

- **后端**：Node.js + Express（统一服务，供管理后台和小程序使用）
- **PC端前端（管理后台）**：Vue3
- **小程序**：微信小程序原生框架
- **数据库**：MySQL 8+

## 项目结构

```
├── doc/            # 项目文档
├── sql/            # SQL文件
├── src/            # 源代码
│   ├── backend/    # 后端代码
│   ├── frontend/   # 前端代码
│   └── weapp/      # 小程序代码
├── .gitignore      # Git忽略文件
└── README.md       # 项目说明
```

## 主要功能

### 1. 线索管理
- 新增线索
- 查看线索列表
- 修改线索状态
- 线索详情查看
- 线索跟进记录

### 2. 结算管理
- 新增结算
- 查看结算列表
- 更新结算状态
- 推送结算提醒

### 3. 用户管理
- 新增用户（经纪人）
- 修改用户信息
- 用户状态管理

### 4. 操作日志
- 记录系统操作日志
- 操作日志查询

## 数据库设计

系统包含以下数据表：
- `c_user`：用户表（管理员和经纪人）
- `c_lead`：线索表
- `c_settlement`：结算表
- `c_operation_log`：操作日志表
- `c_dict`：码表/字典表

详细的数据库设计请参考 `doc/3. database-schema.md`。

## 开发规范

- **命名规范**：
  - 文件夹：小写-中划线
  - 变量：camelCase
  - 数据表：snake_case

- **接口路径**：统一使用 `/api/v1/` 前缀

- **提交规范**：
  - 格式：`类型(范围): 描述`
  - 类型：feat, fix, docs, style, refactor, test, chore
  - 范围：模块名称
  - 描述：简洁明了

- **代码风格**：
  - ESLint + Prettier
  - 2空格缩进
  - 单引号

- **错误处理**：
  - 统一的JSON返回格式
  - 错误码定义

- **安全规则**：
  - JWT鉴权
  - 接口分离

- **性能约定**：
  - 接口响应时间
  - 数据库索引

- **部署规则**：
  - Docker部署
  - 日志管理

- **API版本管理**：
  - 统一前缀

- **测试与质量保证**：
  - 单元测试覆盖率
  - ESLint检查

- **文档维护规则**：
  - 当数据表结构发生变化时，必须更新 `doc/3. database-schema.md` 文件
  - 当码表（c_dict）数据发生变化时，必须更新 `sql/初始化码表.sql` 文件
  - 当数据表结构或业务逻辑发生变化时，必须更新 `sql/常用sql.sql` 文件
  - 所有文档修改都应该保持一致性，确保不同文档之间的信息不冲突
  - 文档更新后，相关的SQL文件也应该同步更新
  - 新增功能或修改功能时，应同时更新相关的文档

详细的开发规范请参考 `doc/2. rules.md`。

## 安装和运行

### 后端安装

1. 进入后端目录
2. 安装依赖：`npm install`
3. 配置环境变量（复制 `.env.example` 为 `.env` 并修改）
4. 启动服务：`npm start`

### 前端安装

1. 进入前端目录
2. 安装依赖：`npm install`
3. 启动开发服务器：`npm run dev`
4. 构建生产版本：`npm run build`

### 小程序安装

1. 进入小程序目录
2. 使用微信开发者工具打开项目
3. 安装依赖：`npm install`
4. 构建项目：`npm run build`
5. 预览或发布小程序
